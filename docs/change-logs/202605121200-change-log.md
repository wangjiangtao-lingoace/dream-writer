# 变更日志

## 变更原因
用户要求完成 P1 功能补全和 P0 代码优化，以完善自动创作小说流程。

## 修改点

### P1 功能补全

1. **后端 Character CRUD 接口**
   - 新增 `server/src/routes/characters.ts`
   - 实现人物卡的增删改查接口
   - 支持按小说 ID 查询人物列表

2. **后端 Worldview CRUD 接口**
   - 新增 `server/src/routes/worldviews.ts`
   - 实现世界观的增删改查接口

3. **后端 KnowledgeAsset CRUD 接口**
   - 新增 `server/src/routes/knowledgeAssets.ts`
   - 实现知识资产的增删改查接口
   - 支持全局和按小说两种查询方式

4. **前端人物卡管理页面**
   - 新增 `client/src/components/CharacterCard.tsx`
   - 实现人物卡的创建、编辑、删除功能
   - 支持记录人物的角色、身份、动机、外貌、背景、关系等信息

5. **前端世界观管理页面**
   - 新增 `client/src/components/WorldviewEditor.tsx`
   - 实现世界观的创建、编辑、删除功能
   - 支持记录世界的规则、地理、势力、历史等信息

6. **前端知识库页面接入真实数据**
   - 新增 `client/src/components/KnowledgeHub.tsx`
   - 替换原来的空壳页面，接入 KnowledgeAsset API
   - 支持创建、编辑、删除知识资产
   - 支持按分类和标签管理

7. **补全 GenerationLog 记录**
   - 修改 `server/src/routes/novels.ts`
   - 在章节生成时记录生成日志
   - 记录成功/失败状态和耗时

### P0 代码优化

1. **拆分 Home.tsx 为独立组件**
   - 将人物卡、世界观、知识库功能拆分为独立组件
   - 减少 Home.tsx 文件大小，提高可维护性

2. **移除重复导航**
   - 移除 `mode-tabs` 重复导航
   - 保留顶部导航栏，新增人物和世界观入口

3. **提取公共 API 工具函数**
   - 新增 `client/src/lib/api.ts`
   - 统一 API 调用方式

- 未引入新的外部依赖，降低风险
