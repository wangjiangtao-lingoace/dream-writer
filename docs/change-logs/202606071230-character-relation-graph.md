# 变更记录：人物关系图可视化功能

## 变更时间
2026-06-07 12:30

## 变更原因
用户需求：实现人物关系图的可视化展示功能，便于直观查看和管理小说中的人物关系网络。系统已有 CharacterRelation 表存储关系数据，但缺少前端可视化界面。

## 修改点

### 1. 后端 API 扩展
- **文件**：`server/src/routes/characters.ts`
- **内容**：新增 `GET /api/characters/relations/:novelId` 端点，用于获取指定小说的所有人物关系数据

### 2. 前端依赖
- **操作**：安装 `reactflow` 库（v11.11.4）
- **用途**：提供图可视化能力（节点、边、交互、布局）

### 3. 关系图组件
- **文件**：`client/src/components/CharacterRelationGraph.tsx`（新建）
- **功能**：
  - 使用 React Flow 渲染人物关系图
  - 节点样式：根据角色定位（主角/反派/配角）显示不同颜色和边框
  - 边样式：根据关系类型（师徒/敌对/盟友/暧昧/恋人/亲属/竞争）显示不同颜色和样式
  - 圆形布局算法：自动计算节点位置
  - 交互功能：点击节点查看角色详情，点击边查看关系详情
  - 右侧详情面板：显示选中的角色或关系的完整信息
  - 左上角图例：展示各种关系类型的说明
  - 支持缩放、拖拽、平移等操作

### 4. 类型定义扩展
- **文件**：`client/src/components/workspace/types.ts`
- **内容**：在 `WorkspaceTab` 类型中新增 `"relations"` 选项

### 5. 工作台集成
- **文件**：`client/src/pages/NovelWorkspace.tsx`
- **修改**：
  - 导入 `CharacterRelationGraph` 组件
  - 在"创作资产"分组中新增"关系图"tab（位于"人物"和"世界观"之间）
  - 在 `renderContent()` 函数中添加 `case "relations"` 渲染逻辑

## 技术细节

### 节点样式映射
- 主角：金色边框 (#f59e0b)，米黄背景
- 反派：红色边框 (#ef4444)，浅红背景
- 配角：蓝色边框 (#3b82f6)，浅蓝背景
- 其他：灰色边框 (#94a3b8)，浅灰背景

### 边样式映射
- 盟友/师徒/亲属：实线，不同颜色
- 敌对/竞争：虚线
- 暧昧/恋人：粉色，带动画效果

### 布局算法
- 圆形布局：节点沿圆周均匀分布
- 半径动态计算：`Math.max(200, total * 30)`，适应不同数量的角色

## 风险说明

### 潜在问题
1. **性能**：角色数超过 20 个时，渲染可能变慢（React Flow 的已知限制）
2. **数据匹配**：CharacterRelation 表存储的是角色名（name），需转换为 ID，如果角色名重复可能导致匹配错误
3. **关系方向**：当前关系可能是双向的（A→B 和 B→A 都存在），可能导致重复显示边

### 边界情况
1. **空数据**：无角色时显示提示文本"暂无人物数据"
2. **缺失关系**：角色存在但无关系时，仍显示所有角色节点
3. **关系无效**：关系指向的角色不存在时，在控制台输出警告并跳过该关系

### 回退方案
- 如需回退，移除 `CharacterRelationGraph.tsx` 文件
- 恢复 `NovelWorkspace.tsx` 和 `types.ts` 的修改
- 卸载 `reactflow` 依赖

## 后续优化建议

1. **力导向布局**：对于复杂关系网络，圆形布局可能不够直观，可考虑引入力导向算法
2. **筛选功能**：添加关系类型筛选器，只显示特定类型的关系
3. **时间轴**：根据 startChapter/endChapter 添加时间轴滑块，查看不同章节的关系变化
4. **编辑功能**：支持在图上直接添加/修改/删除关系
5. **导出功能**：支持导出为图片或 PDF
6. **自动布局优化**：检测孤立节点组并优化布局
