# 变更记录

## 变更原因
基于 UI/UX 深度审核报告，修复 P0（关键反馈）、P1（交互优化）、P2（无障碍/一致性）三个优先级的问题。

## 修改点

### P0：关键反馈缺失（4 项）

#### P0-1：PipelinePage 操作反馈
- `client/src/pages/PipelinePage.tsx`
  - 5 个 mutation handler（pause/resume/confirm/regenerate/saveUserContent）的 catch 块添加 `toast.error()`
  - 对应成功路径添加 `toast.success()`
  - 新增 `pipelineActionLoading` 状态，暂停/恢复按钮添加 loading 态

#### P0-2：NovelWorkspace 自动保存失败提示
- `client/src/pages/NovelWorkspace.tsx`
  - 自动保存 catch 块添加 `toast.error("自动保存失败，请检查网络后手动保存")`
  - 创建章节 catch 块添加 `toast.error("创建章节失败")`

#### P0-3：按钮 loading 态
- `client/src/components/workspace/DashboardPanel.tsx`
  - 新增 `actionLoading` 状态，启动按钮显示"启动中..."
  - 所有按钮在任意操作进行时禁用
- `client/src/pages/BookShelf.tsx`
  - 新增 `deleting` 状态，删除确认按钮显示"删除中..."
- `client/src/pages/Settings.tsx`
  - 新增 `testingId` 状态，测试连接按钮显示"测试中..."

#### P0-4：Settings 删除确认统一
- `client/src/pages/Settings.tsx`
  - `window.confirm()` 替换为 `ConfirmDialog` 组件，与全局风格一致
