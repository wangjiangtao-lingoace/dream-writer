# 变更记录：搜索结果质量提升

## 变更原因

Phase 1（数据源突破）Task 3：提升搜索结果的置信度评分准确性、扩展验证源域名覆盖范围、改进内容提取质量。

## 修改点

### A. 扩展验证源域名（`isVerifiedSourceUrl()`）

在 `search.ts` 中新增三级域名验证体系：
- **Tier 1（高权重正版站）**：qidian.com, jjwxc.net, zongheng.com, 17k.com, ciweimao.com, qimao.com, faloo.com, shuqi.com
- **Tier 2（资讯/社区站）**：zhihu.com, douban.com, baike.baidu.com
- **Tier 3（抓取/聚合站）**：guazixs.cn, lrts.me, fanqienovel.com, sogou.com

新增 `getVerifiedSourceTier(url)` 返回站点等级，`isVerifiedSourceUrl(url)` 判断是否为验证源。Tier 1 站点结果置信度 +0.1，Tier 2 +0.05。

### B. 改进置信度评分（`scoreBlock()` + `calculateConfidence()`）

- 过短内容（<30字）惩罚 -5，过长内容（>3000字）惩罚 -5
- 结构化内容奖励：「作者」「简介」「主角」同时出现 +15
- 叙事动词奖励：检测「说道」「笑道」「突然」等叙事动词，每个 +2
- 章节目录惩罚：检测到 3+ 个章回模式时 -20
- 小说站点特定容器（class="intro"/"bookinfo" 等）加分 +20
- `calculateConfidence()` 增加：过短简介低分、结构化关键词奖励、叙事动词奖励、章节目录惩罚

### C. 改进内容提取（`extractSynopsis()`）

- 新增 `extractMetaSynopsis()`：从 `<meta name="description">` 和 `<meta property="og:description">` 提取高置信度简介
- 新增 `extractNovelSiteSynopsis()`：识别 `<div class="intro">`, `<div class="bookinfo">` 等常见小说站容器
- 简介提取优先级：meta description > 小说站容器 > "简介"关键词块 > 含标题的高分块 > 最高分块
- 新增 `NARRATIVE_VERBS` 常量和 `NOVEL_SITE_SELECTORS` 常量

## 风险说明

- `extractTextBlocks()` 正则对嵌套同名标签（如 div 嵌套 div）的处理存在已知局限，但在当前小说站点场景下影响可控
- meta description 提取依赖站点规范性，部分站点可能不提供或提供垃圾 description
- 章节目录惩罚可能误伤包含多个章节引用的正常简介（边界情况）

## 验证情况

- TypeScript 类型检查通过（`pnpm typecheck:server` + `pnpm typecheck:client`）
- 代码逻辑自审：变更范围限定在评分和提取逻辑，不影响现有路由和数据流
