# 变更记录

## 变更原因
实现通用搜索API集成，替换原有的脆弱直接抓取方式，提供更可靠的搜索能力。

## 修改点
1. 新增 SearchProviderService - 搜索Provider抽象层
   - 支持 Bing Web Search API（首选，1000次/月免费）
   - 支持 Google Custom Search API（备选，100次/天免费）
   - 保留 SogouScrapingProvider 作为最后手段
   - 按优先级自动降级，优雅处理Provider错误

2. 新增 ContentExtractionService - 内容提取服务
   - 使用可读性启发式算法提取网页主要内容
   - 按文本块评分：长度、中文字符密度、小说相关关键词
   - 移除样板内容（页头、页脚、导航、广告）
   - 返回结构化 `{ title, author, synopsis, bodyText }`

3. 修改 search.ts 路由 - 集成新服务
   - 替换原有硬编码5个站点的抓取方式
   - 新流程：SearchProviderService.search → ContentExtractionService.extract → 验证质量 → 返回最佳结果
   - 保留已知来源URL作为降级方案
   - 保持API响应格式兼容

4. 更新 .env.example - 添加搜索配置
   - SEARCH_PROVIDER_PRIORITY=bing,google,sogou
   - BING_SEARCH_API_KEY
   - GOOGLE_SEARCH_API_KEY
   - GOOGLE_SEARCH_CX

## 文件列表
- `server/src/services/SearchProviderService.ts` (新建) - 搜索Provider抽象层
- `server/src/services/ContentExtractionService.ts` (新建) - 内容提取服务
- `server/src/routes/search.ts` (修改) - 集成新服务，替换原有抓取逻辑
- `server/.env.example` (修改) - 添加搜索相关环境变量

## 风险说明
1. **API配额风险**：Bing和Google都有免费额度限制，超限后会自动降级到下一个Provider
2. **网络依赖**：所有Provider都需要网络访问，网络不稳定时可能全部失败
3. **兼容性**：保持了原有API响应格式，前端无需修改
