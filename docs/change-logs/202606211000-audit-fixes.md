# 变更记录 202606211000

## 变更原因
基于 202606201800 业务流程审查报告，修复 P0/P1/P2 级别的业务流程问题。审查从产品经理、业务架构师、最终用户三个视角出发，覆盖业务流程、异常流程、运营流程和数据流转。

## 修改点

### 1. P0: 服务器重启恢复（app.ts）
- 服务器启动时扫描 `status="running"` 的 PipelineJob
- 自动标记为 `error`，附带提示信息"服务器重启，流程自动中断。请点击「恢复」继续"
- 用户可通过「恢复」按钮从断点继续

### 2. P0: Pipeline 失败阶段重试（PipelineService.ts）
- `resumePipeline` 方法增加对 `status="error"` 的处理
- 根据 `currentPhase` 跳转到对应执行函数恢复执行
- writing 阶段计算已完成章节数，从断点继续
- 增加 `retryCount` 计数

### 3. P0: 成本控制 token 预估（PipelineConfigModal.tsx）
- 在配置模态框中添加 token 预估展示区域
- 实时计算：总 tokens = 卷数 × 每卷章数 × (目标字数 × 2.5 + 1500) + 50K 规划开销
- 展示总 tokens、总章数、tokens/章三个指标
- 预估随用户选择的预设或自定义参数实时更新

### 4. P0: 章节生成失败内容保存（writingPhase.ts）
- 章节生成失败时，查找已创建的 Chapter 记录
- 如果章节内容为空，将 `summary` 更新为 `[生成失败] {错误信息}`，`status` 设为 `"failed"`
- 避免章节记录创建后因生成失败而留空

### 5. P1: 伏笔/钩子生命周期检测（consistencyPhase.ts）
- 在一致性校验阶段新增 `detectForeshadowResolutions` 和 `detectHookResolutions`
- 扫描已写完章节内容，匹配伏笔/钩子标题
- 匹配到则自动更新状态为 `paid_off`/`resolved` 并记录回收章节
- 与 LLM 后处理中的检测形成互补

### 6. P1: 一致性修复建议回写（consistencyPhase.ts）
- `attemptAutoFix` 函数将 LLM 生成的修复建议写回 `ConsistencyIssue` 的 `suggestion` 字段
- 用户可在 ConsistencyPanel 中直接查看修复建议

### 7. P1: 全阶段进度推送（pipeline.ts）
- SSE 流式端点扩展支持非 writing 阶段的进度推送
- 检查 `phaseResult` 表变化，推送阶段级进度帧
- 检测 `currentPhase` 变化时推送 `phase_change` 帧
- writing 阶段保持现有章节级推送逻辑不变

### 8. P1: 暂停即时生效（writingPhase.ts）
- 每章开始前查询 PipelineJob 最新状态
- 若 `status="paused"` 或 `status="error"`，立即 break 退出循环
- 当前章节的 LLM 调用仍会完成，但下一章不会再启动

### 9. P2: 错误信息友好化（PipelinePage.tsx）
- 添加 `translatePipelineError` 函数，翻译常见技术错误为用户可读提示
- 覆盖：服务器重启、运行冲突、LLM 失败、超时、频率限制、额度不足、Key 无效
- 进度区域当 Pipeline 失败时展示红色提示框

### 10. P2: 工作台状态 URL 同步（NovelWorkspace.tsx）
- `activeChapterId` 从 URL query 参数 `?chapter=xxx` 读取
- 章节切换时自动同步到 URL，刷新页面后不丢失

## 优化效果

| 问题 | 优先级 | 修复前 | 修复后 |
|------|--------|--------|--------|
| 服务器重启 | P0 | Pipeline 永久卡死 | 自动标记 error，可恢复 |
| 失败重试 | P0 | 只能全流程重跑 | 从失败阶段断点恢复 |
| 成本控制 | P0 | 无预估，无感知 | 实时展示 token 预估 |
| 内容丢失 | P0 | 失败章节留空 | 保存失败信息到 summary |
| 伏笔断裂 | P1 | 状态永不更新 | 一致性校验时自动检测 |
| 修复建议 | P1 | 只存 PhaseResult | 回写到 ConsistencyIssue |
| 进度展示 | P1 | 前 4 阶段无反馈 | 全阶段 SSE 进度推送 |
| 暂停延迟 | P1 | 当前章节完成后才停 | 下一章开始前检查并停止 |
| 错误信息 | P2 | 原始技术错误 | 用户友好的中文提示 |
| 状态持久化 | P2 | 刷新丢失 | URL 同步 activeChapterId |
