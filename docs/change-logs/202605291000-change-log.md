# 变更记录

## 变更原因
工作台展示大量英文字段名（如 `assets/characters`、pipeline status 等），JSON 输入框对非技术用户不友好。需要将英文字段翻译为中文，将 JSON 编辑改为结构化展示/表单。

## 修改点

### Phase 1: 扩展 translate.ts 翻译覆盖
- `client/src/utils/translate.ts`
  - 新增 `translateHookStatus()` — 钩子状态翻译（planted→已埋设, paid_off→已回收 等）
  - 新增 `translateMainlineStatus()` — 主线状态翻译（active→进行中, completed→已完成 等）
  - 新增 `pipelinePhaseLabels` / `pipelineStepLabels` 常量 — 阶段/步骤中文标签
  - 新增 `translatePipelinePhaseLabel()` / `translatePipelineStepLabel()` 翻译函数
  - 新增 `defaultLabelMap` — 通用字段中文标签（80+ 字段覆盖人物/世界观/大纲/卷纲/章纲等）
  - 新增 `snakeCaseToReadable()` — snake_case 转可读文本 fallback

### Phase 2: 修复状态显示硬编码英文
- `client/src/components/workspace/DashboardPanel.tsx`
  - Pipeline 状态从手动三元判断改为 `translatePipelineStatus()` 调用
- `client/src/components/workspace/HookPanel.tsx`
  - 钩子状态从 `hook.status === "planted" ? "已埋设" : hook.status` 改为 `translateHookStatus()`
- `client/src/components/workspace/MainlinePanel.tsx`
  - 主线状态从 `mainline.status === "active" ? "进行中" : mainline.status` 改为 `translateMainlineStatus()`

### Phase 3: BlueprintViewer 结构化渲染
- `client/src/components/workspace/BlueprintViewer.tsx`
  - 对象数组从 `JSON.stringify(item)` 改为显示关键字段卡片（title/name/description/role/goal 等）
  - 嵌套 key 从显示英文 `subKey` 改为 `labelMap[subKey] || defaultLabelMap[subKey] || snakeCaseToReadable(subKey)`
  - labelMap 合并 `defaultLabelMap` 覆盖更多子字段（人物/世界观/卷纲等）

### Phase 4: PipelinePage 中文化和双模式编辑
- `client/src/pages/PipelinePage.tsx`
  - resultKey 显示从 `phase/step` 英文格式改为 `translatePipelinePhaseLabel(phase) / translatePipelineStepLabel(step)` 中文格式
  - 展开视图从 `<pre>JSON.stringify</pre>` 改为 `<SmartJsonViewer>` 结构化展示
  - 编辑模式改为双模式：默认"结构化视图"（SmartJsonViewer 只读）+ "JSON 编辑"（textarea 高级模式）
  - 本地 phaseLabels/stepLabels 替换为导入的翻译函数
  - 资产类型 `item.assetType` 改为 `translateAssetType()` 中文翻译

### Phase 5: SmartJsonViewer 默认 labelMap + KnowledgeHub
- `client/src/components/SmartJsonViewer.tsx`
  - 内置 `defaultLabelMap` 合并逻辑，无外部 labelMap 时自动显示中文字段名
  - 新增 `resolveLabel()` 函数：外部 labelMap → defaultLabelMap → snakeCaseToReadable 三级 fallback
- `client/src/components/KnowledgeHub.tsx`
  - SmartJsonViewer 调用传入知识库专用 `knowledgeLabelMap`（覆盖设定/规则/力量体系/人物等字段）

## 调用链路
```
translate.ts (翻译工具)
  ← DashboardPanel / HookPanel / MainlinePanel (状态翻译)
  ← BlueprintViewer (字段标签 + 结构化渲染)
  ← PipelinePage (阶段/步骤中文 + 双模式编辑)
  ← SmartJsonViewer (内置 labelMap fallback)
  ← KnowledgeHub (知识库 labelMap)
```

## 风险说明
- 不修改后端接口，不改变数据存储格式
- PipelinePage 编辑模式保留 JSON textarea 作为高级编辑 fallback
- SmartJsonViewer 的 labelMap 合并逻辑：外部传入优先 → 内置 defaultLabelMap → snake_case 可读文本
- 所有翻译函数对未知值返回原始英文，不会导致显示异常

## 验证情况
- TypeScript 全量编译通过（client）
