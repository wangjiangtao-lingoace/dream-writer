# 变更记录：全量规划先行架构

## 变更原因
1. 长篇小说（150+ 章）创作中，如果不在初期把所有骨架规划完毕，后期会出现上下文错乱、钩子冲突、人物特性不一致等问题
2. 原流程中章纲缺少每章的详细规划（出场角色、钩子埋/收、伏笔埋/收、爽点设计），写作阶段缺乏精确指导
3. 主线和钩子在章纲之后单独生成，与章节内容脱节
4. 没有一致性校验环节，无法在动笔前发现逻辑漏洞

## 新流程设计

```
outline（智能分析 + 拆解 + 大纲）→ 暂停，用户审核
  ↓
assets（世界观/人物/风格）→ 暂停，用户审核
  ↓
planning 阶段（分步执行）：
  Step 1: volume_outline（卷纲）→ 暂停，用户审核卷纲
  ↓ 用户确认卷纲后
  Step 2: chapter_outline_vol_1..N（逐卷章纲，含每章详细元数据）
  Step 3: story_arcs（跨卷主线/钩子/伏笔/情绪曲线）
  → 暂停，用户审核全部规划
  ↓
consistency_check（一致性校验）← 新阶段
  → 暂停，用户查看校验报告
  ↓
writing（逐章创作）← 只执行，不规划
```

## 修改点

### 一、后端 PipelineService.ts

#### 1.1 新增核心生成方法
- `generateEnrichedChapterOutlines(...)` — 为指定卷生成含完整元数据的章纲（出场角色、钩子埋/收、伏笔埋/收、爽点、情绪曲线）
- `generateStoryArcs(...)` — 生成跨卷主线、跨卷钩子、情绪曲线总览
- `persistVolumeChapterData(...)` — 持久化单卷富化章纲到 ChapterOutline/Hook/Foreshadow/PleasurePoint/EmotionCurve
- `persistStoryArcs(...)` — 持久化跨卷故事弧线到 Mainline/Hook
- `generateConsistencyCheck(...)` — LLM 一致性校验（钩子/伏笔/角色/主线/情绪/爽点/冲突）
- `buildPlanSummaryForConsistency(...)` — 构建校验所需的规划摘要
- `buildEnrichedChapterBlock(...)` — 构建富化章纲的 prompt 区块
- `buildPreviousVolumeSummary(...)` — 构建前序卷章纲摘要

#### 1.1.1 增强卷纲生成
- `generateVolumeOutline(...)` — 新增 `inspiration` 和 `userHint` 参数，卷纲生成时充分调用创意信息
- 卷纲 prompt 增强：要求输出 keyEvents、turningPoint、climax 字段，每卷规划更完整
- 人物摘要优化：展示角色名+定位+动机，便于 LLM 理解人物关系

#### 1.2 新增阶段执行方法
- `executePlanningPhase_standalone(jobId)` — 卷纲生成阶段：生成卷纲 → 暂停等用户审核
- `executeChapterOutlinesPhase(jobId)` — 章纲生成阶段：用户确认卷纲后，逐卷生成章纲 + 跨卷弧线 → 暂停
- `executeConsistencyCheckPhase(jobId)` — 一致性校验阶段 → 暂停

#### 1.3 修改流程控制方法
- `startPipeline` — 添加 `pipelineVersion: 2`，动态计算 `totalSteps`（默认 5 卷 x 30 章 = 150 章）
- `confirmPhase` — 新增分步流转逻辑：
  - 卷纲确认后触发章纲生成（`executeChapterOutlinesPhase`）
  - 所有章纲+弧线确认后进入一致性校验
- `resumePipeline` — 新增 planning 阶段恢复逻辑：
  - 卷纲已确认但章纲未生成 → 生成章纲
  - 所有规划完成 → 进入一致性校验
- `regenerateStep` — 新增 volume_outline/chapter_outline_vol_N/story_arcs/consistency 步骤处理

#### 1.4 修改写作阶段
- `generateInitialChapterDrafts` — 从 planning 阶段读取富化章纲，构建 `enrichedChaptersMap`
- `generateChapterDraft` — 新增 `enrichedChapter` 参数，注入出场角色、钩子、伏笔、爽点、情绪曲线到 prompt

#### 1.5 向后兼容
- `PipelineConfig` 接口新增 `pipelineVersion` 字段
- 旧流程方法（`executeVolumesPhase`、`executeChapterOutlinePhase`）保留为 legacy

### 二、前端 PipelinePage.tsx

- 更新 `phaseLabels`：新增 `planning: "完整规划"`、`consistency_check: "一致性校验"`
- 更新 `stepLabels`：新增 `volume_outline`、`story_arcs`、`consistency`
- 新增动态步骤名解析 `getStepName`：支持 `chapter_outline_vol_N` 格式
- 更新 `defaultPhases`：新流程 `["outline", "assets", "planning", "consistency_check", "writing"]`
- 更新 `summarizeResult`：新增 `volume_outline`、`chapter_outline_vol_*`、`story_arcs`、`consistency` 的摘要渲染

### 三、前端 NovelForm.tsx

- `volumeCount` 默认值：1 → 5
- `chaptersPerVol` 默认值：20 → 30

## 风险说明
- **Token 溢出**：单卷 30 章富化章纲可能超出输出限制 → 可调整 chaptersPerVolume
- **一致性校验误报**：LLM 可能标记非问题 → 校验结果仅作参考，不阻断流程
- **性能**：150 章需 8 次 LLM 调用，约 80 秒 → 可接受（一次性规划）
- **向后兼容**：进行中的旧 job 会受影响 → pipelineVersion 机制保护
