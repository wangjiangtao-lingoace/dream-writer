# 变更日志

## 变更原因
用户要求将系统从"小说写作平台"改造为"AI自动创作系统"，实现一键生成完整小说的全流程自动化。

## 核心改动

### 1. 数据库扩展
**新增表**
- `PipelineJob` - 流程任务表，存储自动化流程状态
- `PhaseResult` - 阶段结果表，存储每个阶段的生成结果、AI自评、用户反馈

**Novel表扩展**
- 新增 `pipelineJob` 关联

### 2. 后端服务新增

**KnowledgeSearchService.ts** - 知识库检索服务
- 关键词匹配检索通用知识库
- 关键词匹配检索作品知识库
- 构建知识上下文注入AI prompt
- 从文本提取关键词

**PipelineService.ts** - 流程编排服务
- 全流程编排：规划→结构化→创作→质控
- 阶段暂停/确认机制
- AI自评和自动修复
- 人工干预支持（确认/重新生成/使用用户内容）

**Pipeline路由** `/api/pipeline`
- POST `/start` - 启动流程
- GET `/novel/:novelId` - 获取小说流程状态
- GET `/:jobId` - 获取流程详情
- POST `/:jobId/pause` - 暂停流程
- POST `/:jobId/resume` - 恢复流程
- POST `/:jobId/confirm` - 确认阶段结果
- POST `/:jobId/regenerate` - 重新生成某步骤
- POST `/:jobId/user-content` - 使用用户内容

### 3. 前端重新设计

**页面优化**
- `BookShelf.tsx` - 书架页：搜索功能、卡片网格优化、SVG图标
- `CreateWork.tsx` - 创作选择页：简化为两个大卡片
- `NovelForm.tsx` - 作品表单页：左右布局、简化表单
- `NovelWorkspace.tsx` - 作品工作台：分组tab导航、可折叠侧边栏

**新增页面**
- `PipelinePage.tsx` - 流程控制页面
  - 流程进度条
  - 各阶段状态展示（规划→结构→创作→质控）
  - 操作按钮（暂停/恢复/确认/重新生成）
  - 步骤详情展示

**样式系统重构**
- 新增 `variables.css` - CSS变量系统
- 更新 `pages.css` - 页面样式
- 新增 `workbench-modern.css` - 工作台样式

### 4. 工作流改造

**原流程（手动）**
```
手动创建作品 → 手动编辑大纲/人物/世界观 → 手动触发生成 → AI生成正文
```

**新流程（自动）**
```
输入创意 → AI生成大纲 → AI生成世界观/人物/风格 → 暂停确认
→ AI生成卷纲/章纲/主线/钩子 → 暂停确认 → AI逐章生成正文 → 暂停确认
→ AI质量控制 → 完成
```

**关键特性**
- 全自动模式：一键生成完整小说
- 阶段暂停：每个阶段完成后暂停，人工确认
- 人工干预：确认/重新生成/手动编辑
- AI自评：每个阶段生成后AI自评打分
- 自动修复：发现问题自动修复
- 知识库注入：生成时检索相关知识库内容

## 风险说明
- PipelineService依赖LLM调用，需要配置有效的API key
- 自动生成过程是异步的，需要轮询状态
- 知识库检索使用关键词匹配，语义匹配能力有限
- 大纲/世界观/人物的自动生成质量取决于LLM能力
