# v2.5 架构改进补丁（3 项关键优化）

## 变更原因
用户（资深架构师）评审 v2.5 后指出 3 个需要补充的点：
1. P0 里必须加入"上一章结尾"，防止跳章
2. Quality Check 增加"缺章/跳章检查"
3. chapterBeat 增加 mustInclude 和 mustAvoid

## 修改点

### 1. P0 层加入"上一章结尾"
- ContextAssembler 新增 `loadPreviousChapterEnding()` 方法，加载上一章最后 500 字
- prompts/index.ts 的 `buildFullSystemPrompt()` 新增 `previousChapterEnding` 参数
- P0 层新增【上一章结尾】和【本章开头承接要求】，优先级高于 Beat 蓝图

### 2. Quality Check 增加剧情承接检测
- qualityCheck.ts 新增 `detectContinuityIssues()` 函数
- 检测 4 类问题：
  - 时间跳跃（几天后、一周后等）
  - 状态突变（突然、忽然、竟然等）
  - 知识跳跃（原来、其实、事实上等）
  - 跳章检测（与上一章结尾关联度）
- QualityScores 新增 `continuity` 字段（0-10 分）
- 剧情承接评分 < 7 时触发问题报告，< 5 时触发重试

### 3. chapterBeat 增加 mustInclude 和 mustAvoid
- schema.prisma 的 ChapterBeat 模型新增：
  - `mustInclude`: JSON 数组，必须包含的元素
  - `mustAvoid`: JSON 数组，必须避免的元素
- writingPhase.ts 的 Beat 保存逻辑更新，持久化新字段
- writingPhase.ts 的 Beat 蓝图文本生成更新，包含 mustInclude 和 mustAvoid
- generators.ts 的 generateChapterBeats prompt 更新，要求 LLM 输出新字段
- writingPhase.ts 的 generateBeatTemplate 更新，8 种 chapterType 模板都包含 mustInclude 和 mustAvoid

## 风险说明
- 需执行 prisma push 同步数据库（ChapterBeat 表新增 2 个字段）
- 剧情承接检测基于关键词匹配，可能存在误报
- Beat 模板的 mustInclude/mustAvoid 需要在实际使用中验证效果
