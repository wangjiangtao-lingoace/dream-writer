# 变更记录：文件导入支持（PDF/EPUB/TXT）

## 变更原因

Phase 1（数据源突破）Task 4：支持用户上传 PDF/EPUB/TXT 文件作为拆书素材，扩展数据源输入方式。

## 修改点

### A. 新建 TextExtractionService（`server/src/services/TextExtractionService.ts`）

新建文本提取服务，支持三种文件格式：
- `.txt`：直接读取，去除 UTF-8 BOM，规范化换行符（\r\n/\r → \n）
- `.pdf`：使用 `pdf-parse` v2 提取纯文本（PDFParse.getText()）
- `.epub`：使用 `epub2` 遍历 spine 章节，逐章提取 XHTML 内容并去除 HTML 标签

所有输出统一截断到 12000 字符。

### B. 修改上传路由（`server/src/routes/upload.ts`）

新增 `documentUpload` multer 配置：
- 允许的 MIME 类型：`text/plain`、`application/pdf`、`application/epub+zip`
- 文件大小限制：5MB

新增 `POST /api/upload/document` 路由：
- 接受 `document` 字段的文件上传
- 调用 `TextExtractionService.extract()` 提取文本
- 返回 `{ text, filename, charCount }`
- 使用 `try/finally` 确保临时文件在提取后被删除

### C. 前端集成（`client/src/pages/AnalyzeCreate.tsx`）

在 textarea 标签行添加「导入文件」按钮：
- 隐藏的 `<input type="file" accept=".txt,.pdf,.epub" />`
- 点击按钮触发文件选择
- 文件选中后 POST 到 `/api/upload/document`（使用 FormData）
- 返回的文本设置到 `pastedText` 状态
- 若标题为空，自动以文件名（去除扩展名）填充标题
- 上传中显示「提取中...」loading 状态

### D. 新增依赖

- `pdf-parse` v2.4.5（~200KB，纯 JS，服务端 PDF 文本提取）
- `epub2` v3.0.2（服务端 EPUB 解析）

## 风险说明

- PDF 提取质量取决于 PDF 文件本身（扫描件 PDF 无法提取文本）
- EPUB 中的图片章节会被跳过，可能导致内容不完整
- 文件上传大小限制 5MB，对于大型书籍文件可能不够
- 临时文件清理使用 `finally` 块，即使提取失败也会删除
