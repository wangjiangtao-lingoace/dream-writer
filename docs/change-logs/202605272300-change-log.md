# 变更记录 2026-05-27 23:00

## 变更原因

执行 10 项系统优化计划（P0/P1/P2 三阶段），提升 LLM 层健壮性、代码可维护性和用户体验。

## 修改点

### P0：基础加固（4 项）

1. **LlmError 错误处理** — 新增 `LlmError` 类和 `completeTextOrThrow()` 方法，`completeText()` 保持向后兼容
2. **LLM 重试机制** — 新增 `withRetry()` 工具函数，指数退避 + jitter，支持 429/500/502/503 和网络错误重试
3. **JSON 解析统一** — 新增 `parseLlmJson()` 工具，替换 PipelineService 和 ImitationPlanService 中的重复实现
4. **Error Boundary** — 新增 `ErrorBoundary` 组件包裹 NovelWorkspace，防止渲染错误崩溃整个页面

### P1：架构优化（3 项）

5. **PipelineService 拆分** — 3,532 行单体拆为 11 个阶段模块 + 704 行编排层
6. **NovelWorkspace 拆分** — 3,089 行单体拆为 9 个子组件 + 676 行布局壳
7. **BookAnalysis 并行化** — 8 个分区分析从串行改为并发（concurrency=4），~4x 提速

### P2：体验增强（3 项）

8. **清理死代码** — 删除 KnowledgeSearchService（93 行，RAG 替代后无引用）
9. **AI-smell 扩展** — 检测模式从 31 条扩展到 92 条，覆盖 7 个类别
10. **Pipeline 流式输出** — 新增 SSE 端点 `GET /api/pipeline/:jobId/stream`，实时推送写作进度
