# AI 写作技能调研与适配分析

## 调研来源

基于 GitHub ai-writing、story-generation 话题，以及 proselint、write-good、textstat、vale 等开源项目的深度分析。

---

## 一、可融入系统的技能清单

### 1. 反 AI 味检测增强（Anti-Slop Detection）

**来源**：GitHub ai-writing 话题中多个项目
**核心思路**：
- **模式审计**：检测并重写 AI 写作模式，而非仅标记词汇
- **风格注入**：让文本更锐利、类型感知，有具体锚点
- **强制人类视角**：注入人类观点和细节

**适配建议**：
- 当前系统的 `aiSmellWords.ts` 仅做词汇级检测，应升级为**模式级检测**
- 参考 write-good 的 8 类检测：被动语态、词汇幻觉、弱化副词、陈词滥调、冗长短语
- 新增**句式单调性检测**：连续 3 句相同主谓结构 = AI 味
- 新增**情感平坦度检测**：连续 5 段情绪强度无变化 = AI 味

**优先级**：P1（当前 AI 味词表已过度激进，需先精简再增强）

### 2. 状态快照架构（State Snapshot Architecture）

**来源**：GitHub ai-writing 话题中的"15维事实快照"项目
**核心思路**：
- 每章维护**15维事实快照**（人物状态、地点、时间、情绪、伏笔等）
- 每章生成**12类变更声明**（角色成长、关系变化、世界观揭示等）
- **6道生成门控**：每章生成前检查前置条件

**适配建议**：
- 当前系统的 `StoryState` 已有类似设计，但维度不够完整
- 建议扩展为 **ChapterSnapshot** 模型，每章生成后自动写入
- 快照维度：主角状态、配角状态、当前地点、时间线、情绪基调、活跃伏笔数、未回收钩子数、世界观变化、核心矛盾进展
- 下一章生成时，自动注入前一章快照作为上下文

**优先级**：P1（可显著提升长篇连贯性）

### 3. 因果链管理（Causal Chain Management）

**来源**：dramatica-flow 项目
**核心思路**：
- 每个剧情事件必须有**因→果**关系链
- 伏笔是"因"，回收是"果"
- 主线矛盾是"终极因"，所有事件必须与之关联

**适配建议**：
- 当前系统有 `Hook` 和 `Foreshadow` 模型，但缺少**因果链表**
- 新增 `CausalLink` 模型：`{fromEvent, toEvent, relationType, chapter}`
- 在章纲生成时，要求 LLM 输出因果关系
- 在一致性检查时，验证每个事件是否有"因"

**优先级**：P2（架构性改进，需较大改动）

### 4. 多 Agent 协作写作（Multi-Agent Writing）

**来源**：GitHub story-generation 话题
**核心思路**：
- **世界构建 Agent**：负责世界观、设定
- **大纲 Agent**：负责剧情结构、节奏
- **写作 Agent**：负责正文生成
- **评审 Agent**：负责质量检测、一致性校验
- **编辑 Agent**：负责润色、去 AI 味

**适配建议**：
- 当前系统的 Pipeline 阶段（outline → assets → planning → consistency_check → writing）已是多阶段，但每个阶段用同一个 LLM
- 建议为不同阶段配置**不同的 system prompt 和 temperature**
- 评审阶段用低 temperature (0.3)，写作用高 temperature (0.7-0.8)
- 新增**编辑 Agent**：写作完成后，用专门的 prompt 做去 AI 味润色

**优先级**：P1（可立即实施，改动较小）

### 5. 可读性评分（Readability Scoring）

**来源**：textstat 项目
**核心思路**：
- 多维度可读性评分：句长、词汇复杂度、音节数
- 适合目标读者群的文本难度
- 中文适配：句长、常用词比例、成语密度

**适配建议**：
- 在 `qualityCheck.ts` 中新增**可读性评分维度**
- 中文指标：平均句长（目标 15-25 字）、常用词比例（目标 >80%）、段落长度（目标 100-300 字）
- 评分过低触发重试或润色

**优先级**：P2（可作为质量检测的补充维度）

### 6. 散文风格检查（Prose Style Linting）

**来源**：proselint、write-good、vale 项目
**核心思路**：
- **陈词滥调检测**：识别过度使用的表达
- **弱化词检测**：识别"非常"、"真的"、"极其"等削弱语气的词
- **冗余检测**：识别不必要的重复和冗长短语
- **被动语态检测**：过多被动语态降低可读性

**适配建议**：
- 当前系统的 AI 味检测可参考 write-good 的分类方法
- 新增**弱化词表**："非常"、"真的"、"极其"、"十分"、"相当"
- 新增**冗余模式**："他心里想"→"他想"，"用眼睛看"→"看"
- 新增**陈词滥调表**："不由自主"、"恍然大悟"、"心如刀割"

**优先级**：P1（直接增强 AI 味检测效果）

### 7. 节奏优化（Pacing Optimization）

**来源**：GitHub story-generation 话题中的 pacing 项目
**核心思路**：
- 将剧情节奏作为独立优化目标
- 压抑→释放→压抑的波浪式节奏
- 每 N 章必须有一个小高潮
- 每 M 章必须有一个喘息章

**适配建议**：
- 当前系统有 `PleasurePoint` 和 `EmotionCurve` 模型，但缺少**节奏规则引擎**
- 新增节奏规则：
  - 连续 3 章无爽点 → warning
  - 连续 5 章高强度 → 建议插入喘息章
  - 爽点间隔 < 2 章 → 警告过于密集
  - 爽点间隔 > 8 章 → 警告过于稀疏
- 在章纲规划阶段自动检查节奏

**优先级**：P1（与现有 StoryState 系统互补）

## 二、不适合当前系统的技能

| 技能 | 原因 |
|------|------|
| Dramatica 理论深度集成 | 过于复杂，当前系统已有简化版角色/冲突管理 |
| 本地 LLM 集成 | 当前系统已支持多 provider，无需额外集成 |
| 可视化关系图 | 已有 CharacterRelation 模型，前端可视化是独立功能 |
| 封面生成 | 与写作质量无关，属于发布阶段 |

---

## 三、实施建议

### 立即可做（1-2天）
1. 精简 AI 味词表 + 参考 write-good 分类增强
2. 为不同 Pipeline 阶段配置不同 temperature
3. 新增编辑 Agent（去 AI 味润色阶段）

### 短期（1周）
4. ChapterSnapshot 快照机制
5. 节奏规则引擎
6. 可读性评分维度

### 中期（2周+）
7. 因果链管理
8. 多 Agent 协作架构优化
