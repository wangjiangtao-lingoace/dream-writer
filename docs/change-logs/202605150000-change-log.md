# 变更记录：智能 Pipeline 架构 + 风格约束增强

## 变更原因
1. 用户有两种创作场景：简单灵感（AI 扩展全部）和丰富输入（AI 拆解保留+补充缺失）
2. 原 Pipeline 固定全量生成，用户给了详细大纲/人物/世界观后，AI 反而生成更差的版本
3. 需要智能识别用户输入内容，拆解入库后只生成缺失部分
4. 原风格控制太简单（只有 narrativePov、pacing 等抽象参数），无法有效约束写作
5. 风格数据没有注入章节生成，导致章节草稿不受风格约束

## 新流程设计

```
用户输入（可简单可丰富）
  ↓
Phase 1: analyze（智能分析）→ 识别输入包含哪些内容类型
  ↓
Phase 1: decompose（拆解入库）→ 将已有内容结构化写入资产表
  ↓
Phase 1: outline（大纲生成）→ 增量补充模式生成大纲
  ↓ 暂停，用户审核大纲
Phase 2: assets（基础资产）→ 世界观/人物/风格，跳过已有资产
  ↓ 暂停，用户审核资产
Phase 3: volumes（卷纲规划）→ 生成卷结构和章纲
  ↓ 暂停，用户审核卷章
Phase 4: chapter_outline（章纲规划）→ 细化每章大纲+主线+钩子
  ↓ 暂停，用户审核章纲
Phase 5: writing（章节创作）→ 生成前 1-3 章
```

## 修改点

### 一、后端 PipelineService.ts

#### 1.1 新增 `analyzeInput` 方法
调用 LLM 分析用户输入，判断包含哪些内容类型：
- hasOutline: 是否包含故事结构
- hasCharacters: 是否包含人物设定
- hasWorldview: 是否包含世界观设定
- hasStyle: 是否包含风格描述
- hasVolumes: 是否包含卷结构/章节规划

#### 1.2 新增 `decomposeIntoAssets` 方法
对每个 `hasXxx: true` 的类型，调用 LLM 将对应内容从文本中提取为 JSON 结构，然后写入数据库：
- `decomposeOutline` → 更新 Novel.outline + KnowledgeAsset
- `decomposeCharacters` → 创建 Character 记录
- `decomposeWorldview` → 创建 Worldview 记录
- `decomposeStyle` → 创建 StyleProfile 记录
- `decomposeVolumes` → 创建 Volume + ChapterOutline 记录

#### 1.3 `executeOutlinePhase` → `executeAnalyzePhase`
改造为三步流程：analyze → decompose → outline

#### 1.4 拆分 `executeGenerationPhase` 为三个独立阶段
原 `executeGenerationPhase` 拆分为：
- `executeAssetsPhase` → 世界观/人物/风格生成（跳过已有资产）
- `executeVolumesPhase` → 卷结构和章纲生成
- `executeChapterOutlinePhase` → 细化章纲 + 主线 + 钩子

每个阶段完成后暂停，等待用户审核确认。

#### 1.5 更新 `confirmPhase` 流程
新流程：outline → assets → volumes → chapter_outline → writing

#### 1.6 更新 `resumePipeline` 和 `regenerateStep`
适配新的阶段划分和流转逻辑。

### 二、前端 NovelForm.tsx

- 移除 Step 2（创作规划），简化为单步表单
- 灵感 textarea 从 6 行扩大到 15 行
- 添加提示文字：「可以只写一小段创意，也可以粘贴完整的大纲、人物设定、世界观等内容。AI 会自动识别并拆解入库。」

### 三、前端 PipelinePage.tsx

- 新增 "智能分析" 和 "拆解入库" 步骤标签
- 分析结果展示：检测到的内容类型（✓/✗）和分析摘要
- 拆解结果展示：已拆解的内容和待生成的内容

### 四、风格约束增强

#### 4.1 增强风格结构
原风格只有抽象参数（narrativePov、pacing 等），新增以下具体约束：
- `toneAndAtmosphere` — 整体基调与氛围（如「表面轻松搞笑，底层有暗流涌动」）
- `emotionalRhythm` — 情绪节奏（如「三章一小高潮，十章一大高潮」）
- `contrastPatterns` — 反差设计（如「强弱反差」「笑中带泪」）
- `humorStyle` — 幽默方式（如「毒舌吐槽型」「冷幽默」）
- `tensionTechniques` — 紧张感制造技巧（如「信息不对称」「倒计时」）
- `suspenseTechniques` — 悬念技巧（如「每章末留未解问题」）
- `sentenceRhythm` — 句式节奏（如「短句为主制造紧张」）
- `dialogueStyle` — 对话风格（如「简洁有力，潜台词丰富」）
- `chapterOpeningStyle` / `chapterEndingStyle` — 开篇/收尾方式
- `writingRules` — 具体写作规则列表
- `avoidList` — 必须避免的写法列表

#### 4.2 风格注入章节生成
- `generateChapterDraft` 新增 `style` 参数
- 在 prompt 中添加「风格约束 — 必须严格遵守」区块
- `generateInitialChapterDrafts` 从 StyleProfile 读取增强风格并传入

#### 4.3 持久化策略
- 增强风格字段存储在 StyleProfile.customRules JSON 中，无需 schema 迁移
- `persistGeneratedAssets` 将增强字段序列化到 customRules
- `generateInitialChapterDrafts` 从 customRules 反序列化读取
