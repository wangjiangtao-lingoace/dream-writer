# 变更记录 - 2026-05-14 02:40

## 变更原因
`LlmInvokeService.resolveModelConfig()` 中 Anthropic provider 被硬编码返回 `null`，导致配置了 API Key 后仍无法使用 Anthropic 模型。

## 修改点
删除 `resolveModelConfig()` 中对 Anthropic 的硬编码拦截（`if (provider === "anthropic") { return null; }`），使其走通用配置解析路径。

通用路径已通过 `providers.ts` 中的辅助函数（`getProviderEnvApiKey`、`getProviderEnvBaseUrl`、`getProviderEnvModel`、`getProviderDefaultBaseUrl`、`providerRequiresApiKey`）完整支持 Anthropic，无需额外实现。

## 文件列表
| 文件 | 变更说明 |
|------|----------|
| `server/src/services/llm/LlmInvokeService.ts` | 删除第 41-43 行 Anthropic 硬编码 `return null`，恢复通用 provider 解析流程 |

## 风险说明
- **风险极低**：仅删除 3 行死代码，不引入新逻辑。
- Anthropic 现在通过通用路径解析，依赖环境变量 `ANTHROPIC_API_KEY`（必需）、`ANTHROPIC_MODEL`（可选）、`ANTHROPIC_BASE_URL`（可选）。
- 若未配置 `ANTHROPIC_API_KEY`，`providerRequiresApiKey("anthropic")` 返回 `true`，会正常返回 `null`，行为与之前一致。
