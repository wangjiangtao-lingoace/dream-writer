# 变更日志：Plate.js v52 富文本编辑器集成

## 变更原因

将 RichTextEditor 从原生 contentEditable + document.execCommand 实现替换为 Plate.js v52 富文本编辑器框架，提供真正的富文本编辑能力（粗体、标题、伏笔标记、角色引用），同时保持存储层纯文本不变。

## 修改点

1. **新增序列化工具**：创建 `plateSerializer.ts`，实现 Plate Value <-> 纯文本的双向序列化/反序列化
2. **重写编辑器组件**：使用 Plate.js v52 的 `Plate`、`PlateContent`、`PlateContainer` 替换原生 contentEditable
3. **插件系统**：定义 3 个自定义插件（ForeshadowPlugin、CharacterRefPlugin、HeadingPlugin），支持富文本标记
4. **工具栏功能**：6 个按钮全部可用（粗体/标题/伏笔标记/角色引用/去AI味/增强压迫）

## 文件列表

| 文件 | 变更说明 |
|------|----------|
| `client/src/utils/plateSerializer.ts` | **新增** - Plate Value 与纯文本双向序列化工具 |
| `client/src/components/workspace/RichTextEditor.tsx` | **重写** - 从 contentEditable 替换为 Plate.js v52 |

## 技术细节

- **导入路径**：使用 `platejs/react`（React 组件和 hooks），类型从 `platejs` 导入
- **API 适配**：Plate.js v52 使用 `createPlateEditor({ value })` 设置初始值（非 `initialValue`），`Plate.onChange` 接收 `{ editor, value }`
- **Props 兼容**：`RichTextEditorProps` 接口保持不变（`content: string, onChange: (value: string) => void`）
- **后端零改动**：数据库 Chapter.content 仍为 String 纯文本

## 风险说明

- **序列化一致性**：serialize -> deserialize 会丢失富文本格式（粗体、标记等），仅保留纯文本内容
- **首次加载**：编辑器仅在组件首次挂载时从 content 初始化（useMemo deps 为空数组），后续 content 变化不重新初始化
- **Plate.js 版本依赖**：依赖 platejs v52.3.21，升级需注意 API 变化
