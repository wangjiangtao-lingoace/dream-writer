# 变更记录

## 变更原因
用户要求：启动后首页先检查是否有 API Key，有的话直接启动，没有的话在首页直接让用户选择模型并输入 API Key，而不是跳转到设置页面。

## 修改点

### 1. BookShelf.tsx — 首页内嵌配置表单
- `client/src/pages/BookShelf.tsx`
  - 将原来的 banner（链接到 /settings）替换为内嵌配置表单
  - 用户可在首页直接选择 AI 提供商、模型、输入 API Key
  - 复用 Settings 页面的 PROVIDERS 列表和 `useCreateConfig()` hook
  - Provider 和 Model 下拉联动
  - 配置成功后自动刷新，表单消失，显示正常书架

### 2. bookshelf.css — 配置表单样式
- `client/src/styles/pages/bookshelf.css`
  - 添加 `.bookshelf-setup` 系列样式
  - 表单两列布局（提供商 + 模型）
  - 错误提示、loading 状态样式

### 3. start.sh — 启动后自动打开浏览器
- `scripts/start.sh`
  - macOS 下启动后自动检测前端端口就绪，然后 `open` 浏览器
  - 等待最多 30 秒，超时不影响正常运行

## 验证情况
- 无 API Key 配置时，首页显示内嵌配置表单
- 选择 provider → model 自动联动
- 填写 API Key 并提交，配置成功后表单消失
- 刷新页面，表单不再出现（已有配置）
- 已有配置时，首页直接显示书架
