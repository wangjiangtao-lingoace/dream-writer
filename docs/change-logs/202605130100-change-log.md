# 变更日志

## 变更原因
用户要求优化前端首页，实现简约沉浸式设计，包含书架、创作、知识库三大功能模块。

## 修改点

### 数据库扩展
1. **Novel表新增字段**
   - `coverImage` - 封面图路径

2. **新增Mainline表**
   - 存储作品主线信息
   - 支持标题、描述、排序、状态

3. **新增Hook表**
   - 存储作品钩子信息
   - 支持类型（悬念/伏笔/喜剧）、强度、状态

4. **新增GeneralKnowledge表**
   - 存储通用知识库信息
   - 支持分类、标签

### 后端API新增
1. **文件上传** `/api/upload`
   - POST `/cover` - 上传封面图
   - POST `/file` - 上传通用文件
   - DELETE `/file` - 删除文件
   - GET `/list` - 获取文件列表

2. **主线管理** `/api/novels/:novelId/mainlines`
   - GET `/` - 获取所有主线
   - POST `/` - 创建主线
   - GET `/:id` - 获取单个主线
   - PUT `/:id` - 更新主线
   - DELETE `/:id` - 删除主线

3. **钩子管理** `/api/novels/:novelId/hooks`
   - GET `/` - 获取所有钩子
   - POST `/` - 创建钩子
   - GET `/:id` - 获取单个钩子
   - PUT `/:id` - 更新钩子
   - DELETE `/:id` - 删除钩子
   - PATCH `/batch/status` - 批量更新状态

4. **通用知识库** `/api/general-knowledge`
   - GET `/` - 获取所有知识（支持分类、搜索、分页）
   - POST `/` - 创建知识
   - GET `/:id` - 获取单个知识
   - PUT `/:id` - 更新知识
   - DELETE `/:id` - 删除知识
   - GET `/stats/categories` - 获取分类统计

### 前端路由重构
**新路由结构**
```
/               → BookShelf（书架）
/create         → CreateWork（创作选择）
/create/new     → NovelForm（新建作品）
/create/analyze → NovelForm（拆书入口）
/novel/:id      → NovelWorkspace（作品工作台）
/novel/:id/:tab → NovelWorkspace（带tab定位）
/knowledge      → GeneralKnowledge（通用知识库）
```

### 新增页面组件
1. **BookShelf.tsx** - 书架页
   - 作品卡片网格展示
   - 封面图显示
   - 进度条和状态
   - 最近编辑时间

2. **CreateWork.tsx** - 创作选择页
   - 独立创作/拆书创作选择
   - 功能特性说明

3. **NovelForm.tsx** - 作品表单页
   - 封面图上传
   - 标题、类型、灵感输入
   - 创建作品

4. **NovelWorkspace.tsx** - 作品工作台
   - 12个功能模块tab切换
   - 侧边栏导航
   - 集成现有组件
   - 新增主线/钩子面板

5. **GeneralKnowledge.tsx** - 通用知识库
   - 分类筛选
   - 搜索功能
   - 知识卡片展示
   - 新增/编辑/删除

### 样式优化
