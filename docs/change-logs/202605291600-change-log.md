# 变更记录

## 变更原因
验收检查发现 DashboardPanel.tsx 中续写模式（continue）的 UI 适配存在 5 个问题，需修复以确保三种创作模式（自主创作/拆书/续写）的界面和交互正确。

## 修改点

### DashboardPanel.tsx 续写模式 UI 修复
- `client/src/components/workspace/DashboardPanel.tsx`
  1. **类型定义**：`creationMode` 联合类型新增 `"continue"`
  2. **卡片布局**：新增 continue 模式专用卡片（已有章节/资产/章节/流程），不再错误显示仿写模式卡片
  3. **描述文案**：新增 continue 模式描述"智能续写：已有章节 → 提取资产 → 规划卷纲章纲 → 每个阶段可确认调整 → 继续创作。"
  4. **任务关系图**：新增 continue 模式专用节点（已有章节 → 分析提取 → 章节创作），不再走进仿写模式节点图
  5. **进度计算**：standalone/continue 模式使用 pipeline 实际进度，imitation 模式使用四阶段 25% 计算
  6. **重复按钮**：移除硬编码的"智能续写"按钮（与 nextActions 中的 continue 按钮重复），改为在 nextActions 按钮中为 continue 按钮添加高亮样式
  7. **清理**：移除未使用的 `handleContinue` 函数、`continuing` 状态、`toast` 和 `BlueprintViewer` 导入

## 验证情况
- TypeScript 全量编译通过（client）
- 三种模式 UI 逻辑审查完毕：
  - 自主创作：灵感 → Pipeline（standalone）→ 流水线页面 ✅
  - 拆书仿写：资料 → 拆书 → 仿写方案 → Pipeline（imitation）→ 流水线页面 ✅
  - 智能续写：已有章节 → Pipeline（continue）→ 流水线页面 ✅
