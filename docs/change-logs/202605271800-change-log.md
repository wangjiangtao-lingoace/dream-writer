# 变更记录 2026-05-27 18:00

## 变更原因

实现阶段二 Task 3：知识写入管道（RagIngestService），知识创建/更新/删除时自动切片+嵌入+存储。

## 修改点

### 新建文件
- `server/src/services/RagIngestService.ts` — 知识写入管道核心服务

### 修改文件
- `server/src/routes/knowledgeAssets.ts` — create/update/delete 触发 RAG ingest
- `server/src/routes/memories.ts` — create/update/delete 触发 RAG ingest
- `server/src/routes/generalKnowledge.ts` — create/update/delete 触发 RAG ingest
- `server/src/services/PipelineService.ts` — saveToKnowledgeBase 触发 RAG ingest

### 功能变更

1. **RagIngestService 核心流程**
   - `ingestText()`: 删除旧 chunks → 切片+嵌入 → 写入 RagChunk → 写入 vec0
   - `deleteChunks()`: public 方法，清理 RagChunk + vec0（供路由层删除前调用）
   - per-owner 互斥锁：防止并发 ingest 干扰
   - 失败回滚：写入失败时清理已部分写入的 RagChunk 和 vec0
   - chunk 数量一致性校验

2. **路由集成**
   - 所有 create/update 路由：fire-and-forget 触发 ingest（`.catch(console.error)`）
   - PUT 路由仅在 content 字段变更时触发
   - 所有 delete 路由：删除前清理 RAG 数据

3. **PipelineService 集成**
   - `saveToKnowledgeBase()` 完成后触发 ingest
