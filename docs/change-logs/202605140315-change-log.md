# 变更记录

## 变更原因
优化批量生成章节的性能。原实现使用 `for` 循环串行执行，每章需两次 LLM 调用（正文 + 去 AI 味），生成 20 章需 40 次串行 LLM 调用，耗时极长。

## 修改点
将串行执行改为分批并行执行，并发限制为 3，显著减少批量生成总耗时。

## 文件列表
- `server/src/routes/batchGenerate.ts` — 将逐章串行循环重构为 `Promise.all` 分批并行执行，每批最多 3 章并发

## 风险说明
- 并发限制为 3，不会对 LLM API 造成过大压力
- SQLite 在并发写入时可能出现锁竞争，但 Prisma 的连接池和事务机制可处理此场景
- 各章节生成逻辑相互独立，无顺序依赖，并行执行安全
