# 变更记录

## 变更原因
对首页与工作台的风格一致性和流程顺畅度进行审查，发现并修复 3 个严重问题和 6 个中等问题。主要涉及状态反馈缺失、布局跳变、双层 header、CSS 变量不统一等。

## 修改点

### 状态反馈修复（5 个文件）
- `client/src/pages/BookShelf.tsx` — 加载失败增加 toast.error、删除成功/失败增加 toast 反馈、进度条从硬编码 `/10` 改为基于 chaptersPerVol 的动态计算、删除按钮增加 hover 红色危险样式
- `client/src/pages/NovelForm.tsx` — 创建成功增加 toast.success、跳转 URL 添加 `?new=true` 参数
- `client/src/pages/AnalyzeCreate.tsx` — 创建成功增加 toast.success、跳转 URL 添加 `?new=true` 参数
- `client/src/pages/ImportContinue.tsx` — 跳转 URL 添加 `?new=true` 参数
- `client/src/pages/NovelWorkspace.tsx` — 新增 `useSearchParams`，检测 `?new=true` 时默认进入 write tab；notice timer 改用 useRef 追踪避免竞态

### 布局一致性修复（2 个文件）
- `client/src/components/layout/AppLayout.tsx` — 新增 `isCreatePage` 判断，TopBar 在 create 页面不渲染，消除双层 header
- `client/src/components/workspace/WorkspaceTopBar.tsx` — 品牌区域改为可点击返回，显示小说标题和"返回书架"文字

### CSS 变量统一（3 个文件）
- `client/src/styles/tokens.css` — 新增 `--accent-subtle`、`--accent-border`、`--accent-border-strong` 三个透明度变量
- `client/src/styles/pages/bookshelf.css` — 4 处硬编码 rgba 替换为 CSS 变量
- `client/src/styles/components.css` — 5 处硬编码 rgba 替换为 CSS 变量

## 风险说明
- 所有变更均为 UI 层面，不涉及业务逻辑和后端接口
- `?new=true` 参数仅影响默认 tab 选择，不影响其他功能
- CSS 变量替换保持了完全相同的计算值，视觉效果无变化

## 验证情况
- TypeScript 全量编译通过（shared + server + client）
