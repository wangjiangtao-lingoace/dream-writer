# 变更记录 - 2026-05-14 01:20

## 变更原因
P0-2 创建向导增强：扩展 Novel 表以支持更丰富的小说元数据，并将创建表单重构为两步向导。

## 修改点

### Prisma Schema 扩展
- **Novel 表新增 6 个字段**：
  - `synopsis` (String?) — 故事简介/梗概
  - `targetWordCount` (Int, default 300000) — 目标总字数
  - `chapterWordMin` (Int, default 2000) — 章节最小字数
  - `chapterWordMax` (Int, default 4000) — 章节最大字数
  - `volumeCount` (Int, default 1) — 卷数
  - `chaptersPerVol` (Int, default 20) — 每卷章节数

### 共享类型更新
- **shared/src/types/novel.ts** — Novel 接口新增对应字段，移除重复的 outline 定义

### 后端服务更新
- **NovelService.ts** — CreateNovelInput 和 UpdateNovelInput 接口新增字段
- **novels.ts (路由)** — novelCreateSchema 和 novelUpdateSchema 新增 Zod 验证，路由处理 null → undefined 转换

### 前端重构
- **NovelForm.tsx** — 重构为两步向导：
  - Step 1: 基础信息（标题、类型、灵感、封面）
  - Step 2: 创作规划（简介、目标字数、章节字数范围、卷数、每卷章节数）
  - 新增步骤进度指示器
  - 新增预估字数计算显示

- 数据库需要重新执行 `pnpm prisma:push` 同步 schema（已有数据库会保留数据，新字段使用默认值）
- 新字段均为可选字段，不会影响现有功能
- 向导第一步的"下一步"按钮会验证标题非空
