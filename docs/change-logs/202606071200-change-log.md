# 变更记录 - 2026-06-07 12:00

## 变更原因
全面优化小说创作流程，解决14个关键缺陷，提升长篇小说（150万字+）的创作质量和一致性。

## 修改点

### P0 - 核心修复（5项）

1. **主题防偏系统**
   - 在 ContextAssembler 中注入 Novel.genre 到写作上下文
   - 添加「主题硬约束」段落，明确告知 LLM 当前作品类型
   - 防止都市文写成玄幻、仙侠文混入现代元素

2. **Mainline 注入写作上下文**
   - 新增 `loadMainlines()` 方法，加载当前章节范围内的活跃主线
   - 在写作 prompt 中添加「主线剧情」段落
   - 确保每章写作围绕主线展开

3. **CharacterRelation 注入写作上下文**
   - 新增 `loadCharacterRelations()` 方法，加载本章出场角色之间的关系
   - 在写作 prompt 中添加「角色关系」段落
   - 支持盟友/敌人/师徒/恋人/亲属/竞争等关系类型

4. **StoryState 禁止/允许列表生效**
   - 修改 `loadStoryState()` 加载 forbiddenActions 和 allowedActions
   - 在写作 prompt 中添加「禁止剧情」和「允许剧情」段落
   - 硬约束：禁止列表中的剧情绝对不能出现

5. **爽点冷却硬约束**
   - 修改 `loadStoryState()` 计算有效冷却值
   - 在写作 prompt 中添加「爽点冷却」段落
   - 冷却期内禁止出现金手指、复仇、震撼等高能情节

### P1 - 重要优化（4项）

6. **大神级作家风格模仿系统**
   - 在 `generateStyle()` 中添加 `masterWriterStyle` 字段
   - 根据作品类型自动推荐对应的白金大神风格
   - 在 ContextAssembler 中注入「作家风格模仿」段落

7. **EmotionCurve 注入写作上下文**
   - 新增 `loadEmotionCurve()` 方法，加载当前章节的情绪曲线数据
   - 在写作 prompt 中添加「情绪曲线指导」段落
   - 包含情绪类型、强度、是否高潮/转折/呼吸章

8. **统一 AI 味词汇表**
   - 创建 `aiSmellWords.ts` 共享模块
   - 合并 writingRules.ts 和 StyleService.ts 的两份词汇表
   - 统一为单一来源，消除冗余和不一致

9. **充分利用 styleProfile 完整规则**
   - 修改 `loadStyleCompact()` 加载所有风格维度
   - 包括：叙事视角、时态、句式长度、词汇风格、对话比例、情感强度、幽默程度
   - 在写作 prompt 中生成完整的风格约束段落

### P3 - 安全修复（1项）

10. **更新小米 API Key**
    - 更新 .env.example 中的 MIMO_API_KEY
    - 修复 MiMo baseURL 不一致问题

### P2 - 补充修复（2项）

11. **质量后验添加主题偏离检测**
    - 在 qualityCheck.ts 中添加 `detectGenreDeviation()` 函数
    - 根据 Novel.genre 检查内容中是否出现了不该出现的关键词
    - 支持都市/玄幻/仙侠/言情/悬疑/历史等类型的禁止词表
    - 偏离评分：0=无偏离，1=轻微，2=中等，3=严重（>=3则重试）

12. **后处理自动更新 forbiddenActions/allowedActions**
    - 修改 postProcessing.ts 的 `mergedPostProcessing` 函数
    - 在 LLM prompt 中添加 forbiddenActions 和 allowedActions 字段提取
    - 在 StoryState upsert 中更新这些字段
    - 确保每章写完后自动设置下一章的剧情约束

## 风险说明

- **低风险**：所有修改都是增量添加，不改变现有逻辑
- **向后兼容**：新增字段都有默认值，旧数据不受影响
- **性能影响**：ContextAssembler 新增 3 个数据库查询（mainlines、characterRelations、emotionCurve），但都是轻量查询，影响可忽略
