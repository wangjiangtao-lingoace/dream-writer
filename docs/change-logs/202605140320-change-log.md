# 后端 BYOK 实现变更记录

**变更原因**: 为前端 Settings 页面提供后端 AI 配置 CRUD API 和 BYOK（Bring Your Own Key）支持，使用户可通过自带 API Key 使用 AI 功能。

**分支**: `feat/lightweight-rebuild`

## 修改点

### 1. AIConfig Prisma 模型
- 在 `schema.prisma` 新增 `AIConfig` 模型，字段：id, provider, model, apiKey（AES-256 加密）, baseUrl, isDefault, createdAt, updatedAt
- 已执行 `prisma db push` 同步到 SQLite

### 2. API Key 加密工具
- 新增 `server/src/utils/crypto.ts`
- 使用 AES-256-CBC + scrypt 密钥派生，格式：`salt:iv:ciphertext`（hex 编码）
- 环境变量 `AI_CONFIG_ENCRYPTION_KEY` 控制密钥，未设置时使用开发兜底密钥并打印警告

### 3. AIConfig CRUD 路由
- 新增 `server/src/routes/aiConfig.ts`，注册到 `/api/ai-config`
- `GET /` — 列表（不返回 apiKey）
- `GET /default` — 默认配置（返回解密后的 apiKey，内部使用）
- `POST /` — 创建（加密存储 apiKey）
- `DELETE /:id` — 删除
- `POST /:id/test` — 测试连接（调用 provider 的 /models 端点）

### 4. LlmInvokeService BYOK 优先级
- `resolveModelConfig()` 改为 async，优先级：DB 默认配置 > .env 环境变量
- 所有调用点（completeText/streamText/streamChapterDraft）已更新为 await

## 文件列表

### 新建文件（2 个）
- `server/src/utils/crypto.ts` — AES-256 加密/解密工具
- `server/src/routes/aiConfig.ts` — AI 配置 CRUD 路由

### 修改文件（3 个）
- `server/prisma/schema.prisma` — 新增 AIConfig 模型
- `server/src/app.ts` — 注册 aiConfigRouter
- `server/src/services/llm/LlmInvokeService.ts` — resolveModelConfig 支持 DB 优先

## 风险说明
- 开发环境未设置 `AI_CONFIG_ENCRYPTION_KEY` 时使用兜底密钥，生产环境必须配置
- `GET /default` 端点返回解密后的 apiKey，仅限内部服务调用，不应暴露给前端

## 验证情况
- `pnpm typecheck` — 全部通过
- `pnpm build` — 全部通过
- API 端点手动验证：GET/POST/DELETE /api/ai-config 均正常

# 工作台 UI 重构变更记录

**变更时间**：2026-05-14 03:20
**变更类型**：UI/UX 优化
**变更分支**：feat/lightweight-rebuild

## 变更原因

用户反馈当前工作台存在多个 UI 问题：
1. 章节目录位置不合理，应该在作品下方
2. 创作蓝图直接显示 JSON 数据，不可读
3. 进入工作台后全局侧边栏仍然显示，希望隐藏
4. 拆书分析的 8 个分区显示为小方块，看不清楚
5. 整体结构混乱，需要更清晰简洁的界面
6. 任务关系图未实现

核心需求：小说创作需要大篇幅文字展示，界面应**接口清晰、页面简洁**。

## 修改点

### 1. 布局重构：工作台全屏模式
- 进入工作台后隐藏全局侧边栏和顶部栏，最大化创作空间
- 优化"返回书架"按钮样式，增加文字说明
- 顶部 header 增加分隔线，视觉更清晰

### 2. 章节目录重构：可折叠面板
- 移除左侧独立章节目录栏
- 在 header 下方增加可折叠章节目录面板
- 章节以网格布局展示，带序号和选中状态
- 显示章节数量统计，支持折叠/展开

### 3. 创作蓝图：结构化展示
- 创建 `BlueprintViewer` 组件，解析 JSON 并转换为结构化卡片
- 每个 key 显示为中文标签，value 用格式化文本展示
- 支持展开/折叠长文本，数组和对象嵌套展示
- 保持原有数据完整性，仅改变展示方式

### 4. 拆书分析 8 分区：标签页切换
- 将 8 个分区改为标签页布局
- 左侧显示分区标签导航（带状态图标）
- 右侧显示当前选中分区的完整内容
- 保留"用于仿写"开关和"修改"按钮

### 5. 任务关系图：简化实现
- 在"总控"tab 中添加任务关系图
- 使用卡片+连接线展示任务依赖关系（4 个节点）
- 每个节点显示：序号、名称、状态、进度
- 添加整体进度条，直观显示创作流程完成度

# 工作台 UI 优化变更记录（第二轮）

**变更时间**：2026-05-14 04:00
**变更类型**：UI/UX 优化
**变更分支**：feat/lightweight-rebuild

## 变更原因

用户反馈工作台存在9个UI问题：
1. 章节目录点击没有响应
2. 世界观展示错乱
3. 大纲还是JSON格式
4. 人物关系分析不可用
5. 写作中有英文，点击章节不显示详情
6. 知识库是JSON格式
7. 记忆也是JSON格式
8. 总控中有英文显示
9. 人物类型不全

核心需求：小说创作需要大篇幅文字展示，界面应**接口清晰、页面简洁**，采用**自动为主、人工审核**的工作流模式。

## 修改点

### 1. 创建通用展示组件
- **SmartJsonViewer**：智能JSON展示组件，根据数据类型选择展示方式
  - 简单对象：表格展示
  - 数组：列表展示
  - 复杂嵌套：树形展示
- **翻译工具**：`translate.ts`，将英文状态翻译为中文配合图标

### 2. 修复章节目录点击响应
- 添加 `expandedChapterId` 状态，控制章节目录下方展开的章节详情
- 点击章节后自动切换到写作面板，并在下方展开详情区域
- 详情区域显示：字数统计、状态、内容预览、编辑按钮

### 3. 优化世界观展示
- 创建 **WorldviewViewer** 组件，用于只读展示世界观数据
- 将世界观数据解析为多个卡片：概述、规则、力量体系、地理、派系、历史、文化
- 每个卡片支持展开/折叠，显示完整内容
- 保留 WorldviewEditor 用于编辑场景

### 4. 优化大纲展示
- 创建 **OutlineViewer** 组件，解析JSON并结构化展示
- 支持两种视图模式：结构化视图和原始视图
- 结构化视图使用 SmartJsonViewer 组件展示
- 支持编辑功能，点击编辑按钮切换到编辑模式

### 5. 优化英文显示
- 创建翻译工具函数：
  - `translateChapterStatus()`：翻译章节状态（planned→📋已计划）
  - `translateChapterSource()`：翻译章节来源（manual→✋手动）
  - `translatePipelineStatus()`：翻译Pipeline状态
  - `translatePipelinePhase()`：翻译Pipeline阶段
- 修改写作面板和总控台，使用图标+中文的方式展示

### 6. 扩展人物类型
- 扩展 `ROLE_OPTIONS` 数组，增加更多类型：
  - 网文类型：系统、后宫、小弟、师父
  - 经典叙事类型：旁白、牺牲者、守护者、背叛者、催化剂、信使、变形者、影子
  - 自定义选项：允许用户输入自定义角色类型

### 7. 优化知识库展示
- 修改 **KnowledgeHub** 组件，使用 SmartJsonViewer 展示知识库内容
- 支持嵌套JSON对象的智能展示
- 自动解析JSON并选择最佳展示方式

### 8. 优化记忆展示
- 修改 **MemoryPanel** 组件，使用 SmartJsonViewer 展示记忆内容
- 支持结构化展示，避免直接显示JSON文本
- 保持原有功能不变，仅优化展示方式

### 9. 实现人物关系分析
- 创建 **RelationshipMatrix** 组件，使用表格展示人物关系矩阵
- 行列显示所有人物，交叉处显示关系类型
- 支持点击单元格编辑关系，选择关系类型和添加描述
- 预设10种关系类型：朋友、敌人、师徒、恋人、亲人、同事、对手、盟友、背叛、保护
- 集成到人物管理面板
