# 变更记录

## 变更原因
项目集成功能增多，需要统一启动脚本，让用户一条命令完成首次安装和日常启动。

## 修改点

### 删除 `scripts/setup.sh`
功能已合并到 start.sh，不再需要单独的安装脚本。

### 重写 `scripts/start.sh` — 统一启动脚本
首次安装和日常启动共用一条命令：`bash scripts/start.sh`

8 步自动化流程：
1. 检查运行环境（Node.js >= 20.19, pnpm >= 10.6, macOS/Linux 双平台）
2. 同步代码（项目不存在则自动 clone，已存在则 git pull 拉取更新）
3. 检查端口占用（5173/3000），可选释放
4. 同步依赖（pnpm install，仅在 lock 变化或 node_modules 缺失时）
5. 同步数据库（prisma:generate + prisma:push，仅在 schema 变化时）
6. 构建共享类型包（仅在 shared 源码变化时）
7. 检查环境配置（自动生成 .env + AI_CONFIG_ENCRYPTION_KEY）
8. 启动 pnpm dev

## 验证情况
- bash 语法检查通过
- 实际运行：环境检查、端口检测、Git 检测均正常
