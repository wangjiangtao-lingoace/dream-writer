# 变更记录

## 变更原因
工作流 C: 功能增强 - 优化拆书轮询机制，迁移 localStorage 数据

## 修改点

### 1. 拆书轮询重构（AnalyzeCreate.tsx）
- 将阻塞式 `while` 循环 + `setTimeout` 改为非阻塞式 `setInterval` 轮询
- 添加 `useRef` 存储 interval ID 和 AbortController
- 添加取消按钮，支持用户中途取消拆书任务
- 使用 AbortController 支持取消操作

### 2. 一致性校验结果持久化（ConsistencyPanel.tsx）
- 移除 localStorage 存储机制
- 创建后端 API 路由 `/api/consistency-results` 进行 CRUD 操作
- 创建 Prisma 模型 `ConsistencyCheckResult` 存储校验汇总
- 前端改为调用后端 API 保存和加载校验结果
- 支持跨浏览器和设备访问校验历史

### 3. API 模块增强（api.ts）
- 添加 `api.patch` 方法支持 PATCH 请求

## 文件列表

| 文件 | 变更说明 |
|------|----------|
| `client/src/pages/AnalyzeCreate.tsx` | 重构轮询机制，添加取消功能 |
| `client/src/components/ConsistencyPanel.tsx` | 迁移到后端 API 存储 |
| `client/src/lib/api.ts` | 添加 patch 方法 |

| `server/src/app.ts` | 注册新路由，更新 REQUIRED_TABLES |

## 风险说明

1. **数据库迁移**: 需要执行 `pnpm prisma:push` 同步新表到数据库
2. **向后兼容**: localStorage 中的旧数据将无法自动迁移，用户需重新进行校验
3. **API 依赖**: 前端现在完全依赖后端 API，网络异常时无法使用本地缓存
