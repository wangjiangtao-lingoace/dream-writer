# 变更记录：卷纲/章纲流程修复

## 变更原因
1. 独立章纲生成（非 Pipeline 模式）缺少大纲、世界观、主线、风格等核心上下文，导致 AI 生成质量低于 Pipeline 模式
2. VolumeEditor 组件使用了大量 CSS 类名但无对应样式定义，UI 基本无样式
3. 卷纲/章纲缺少"列表→详情"的二级导航结构
4. 无章纲时缺少显眼的 AI 生成按钮，用户难以发现生成功能

## 修改点

### 1. AIService 章纲生成上下文增强（高优先级）
- `generateChapterOutline`：Prisma 查询补充 worldviews、mainlines、styleProfiles；prompt 增加故事大纲、世界观、主线、风格、前序卷摘要
- `generateChapterOutlinesForVolume`：同样补充世界观和大纲上下文，增强人物信息（含 identity/motivation）

### 2. VolumeEditor UI 重构（中优先级）
- 布局从平铺改为 sidebar（280px 卷列表）+ main（卷详情 + 章纲列表）
- 卷列表：卡片显示卷名、章数 badge、目标摘要，选中态橙色高亮
- 卷详情面板：展示目标、冲突、情绪、地图、结尾钩子、关键事件、转折点
- 章纲列表：可点击打开 Modal 查看详情（目标、冲突、情绪、爽点、钩子、伏笔）
- 空状态：居中显示"AI 生成章纲"主按钮 + "手动新建"次按钮

### 3. CSS 补充
- components.css 追加 section-header、desk-actions、volume-stats 样式

## 文件列表

| 文件 | 说明 |
|------|------|
| `server/src/services/AIService.ts` | 两个章纲生成函数补充世界观/大纲/主线/风格/前序卷上下文 |
| `client/src/components/VolumeEditor.tsx` | 重构为 sidebar+main 布局，添加 Modal 详情、空状态 CTA |
| `client/src/styles/components.css` | 追加 VolumeEditor 相关 CSS |

## 风险说明
- AIService 的 Prisma 查询新增了 worldviews/mainlines/styleProfiles 关联，如果这些表为空不影响生成（均有 fallback）
- VolumeEditor 布局重构为全新代码，需验证原有 CRUD 功能（新建/编辑/删除卷和章纲）不受影响
- 前序卷摘要查询仅取每卷前 5 个章纲标题，控制 prompt 长度
