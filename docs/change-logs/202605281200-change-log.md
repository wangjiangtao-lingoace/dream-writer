# 变更记录

## 变更原因
完成 Dream Writer 创作工作台全面重构计划的 Phase 4（工作台布局重写）和 Phase 5（全局页面主题适配）。将"写作" tab 从旧的 2 列布局升级为 3 列布局（章节侧栏 + 编辑器 + 资产面板 + 顶栏 + 底栏），同时确保全局页面已完全切换为白色主题。

## 修改点

### Phase 4: NovelWorkspace 布局重写
- `client/src/components/workspace/types.ts` — 新增 `ChapterWithVolume`、`WorkspaceData`、`RadarScores`、`AIReview` 四个接口
- `client/src/components/workspace/index.ts` — 新增 8 个组件导出（WorkspaceTopBar、WorkspaceBottomBar、ChapterSidebar、RichTextEditor、ChapterHeaderView、AssetPanel、AIScoreCircle、RadarBar）和 4 个新类型导出
- `client/src/pages/NovelWorkspace.tsx` — 主要变更：
  - "write" tab 渲染新 3 列布局：WorkspaceTopBar + (ChapterSidebar | 编辑器 | AssetPanel) + WorkspaceBottomBar
  - 挂载时调用 `/api/novels/:id/workspace-data` 和 `/api/novels/:id/radar-scores` 获取数据
  - 章节切换时调用 `/api/novels/:id/chapters/:chapterId/ai-review` 获取 AI 评审
  - 编辑器内容变更时 2 秒防抖自动保存
  - 非 write tab 保持原有布局不变
- `client/src/styles/pages/workspace.css` — 新增 `.workspace-write-layout`、`.workspace-write-body`、`.workspace-write-main`、`.workspace-write-main-inner`、`.workspace-write-empty` 五个 CSS 类

### Phase 5: 全局页面主题适配
- `client/src/styles/pages/workspace.css` 第 118 行 — `rgba(249,115,22,0.1)` → `var(--accent-muted)`（旧琥珀色替换为新蓝色主题变量）

## 风险说明
- 后端 workspace API 端点已在 Phase 2 实现，前端会静默降级到默认值
- RichTextEditor 使用 contentEditable 作为过渡方案，Plate.js 集成作为后续任务
- AssetPanel 的 worldviews 暂传空数组，后续可从 workspaceData 加载
- 语义化颜色（绿色成功、红色错误等）保留不变，适用于浅色和深色主题

- TypeScript 全量编译通过（shared + server + client）
- 10 种指定的硬编码暗色主题颜色值已全部清除
