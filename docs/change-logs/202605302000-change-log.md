# 质量全面提升：6 Phase × 15 维度 × 9 分目标

## 变更原因

产品评审诊断出系统综合得分 6.7/10，距"优秀"产品有系统性差距。核心问题：
1. 规划与执行脱节 — 规划阶段产出丰富的结构化数据，写作阶段只靠 prompt 约束，不校验输出
2. 一次性检查 vs 持续监控 — 一致性检查只在规划时运行一次，500 章写作过程中不再检查
3. 续写模式是二等公民 — pipeline 模式有 15 条规则+状态追踪+记忆管理，续写模式全缺
4. 后验缺失 — 整个系统是"生成→保存"，没有"生成→验证→修复"闭环
5. 可观测性不足 — 没有作品级健康仪表盘

## 改动

### Phase 1: Schema 数据模型基础

`server/prisma/schema.prisma`：
- Character 模型增强：新增 speechStyle、arcDetail、lastAppear、appearanceCount、knowledgeScope 字段
- Chapter 模型增强：新增 qualityScore、aiSmellCount、reviewStatus、reviewNotes 字段
- Memory 模型增强：新增 lastAccessedAt、accessCount、lastAccessedChapter 字段
- Foreshadow 模型增强：新增 intensity 字段
- 新增 CharacterRelation 模型（角色关系）
- 新增 StoryStateLog 模型（剧情状态历史快照）
- 新增 ChapterQualityLog 模型（章节质量日志）

### Phase 2: 写作质量后验闭环

`server/src/services/pipeline/writingRules.ts`（新建）：
- 提取公共写作规则模块，统一 15 条铁律
- AI 味词汇和短语列表
- 字数校验函数

`server/src/services/pipeline/qualityCheck.ts`（新建）：
- 章级质量后验函数：正则检测 + 字数校验 + LLM 评分
- 综合判定是否需要重试

`server/src/services/pipeline/writingPhase.ts`：
- 集成质量后验循环（最多重试 2 次）
- 保存质量日志到 ChapterQualityLog 表
- 更新 Chapter 质量字段
- 集成角色知识边界检查（禁忌知识注入写作 prompt）

`server/src/services/ContinuationService.ts`：
- 替换为共享 15 条完整规则
- 集成质量后验
- 添加角色状态更新、记忆管理

### Phase 3: 上下文与记忆增强

`server/src/services/pipeline/contextAssembler.ts`：
- 扩展上下文窗口：每 50 章一个里程碑摘要
- 记忆访问追踪：更新 lastAccessedAt、accessCount
- 世界观上下文扩展：summary/rules 截断从 200 → 300 字

`server/src/services/pipeline/characterKnowledge.ts`（新建）：
- 角色知识边界追踪：从章节内容提取"谁知道了什么"
- 禁忌知识检查：写作前获取角色不应该知道的信息
- 知识摘要：用于上下文组装

`server/src/services/pipeline/postProcessing.ts`：
- StoryState 历史快照：更新前保存到 StoryStateLog
- 角色出场追踪：更新 lastAppear、appearanceCount
- 角色成长线：新增 arcDetail 存储
- 集成角色知识范围更新

### Phase 4: 生命周期与持续一致性

`server/src/services/pipeline/periodicCheck.ts`（新建）：
- 周期性一致性检查（每 10 章）
- 钩子/伏笔逾期告警
- 角色漂移检测（LLM）
- 情绪节奏校验（连续高潮/低谷）

`server/src/services/pipeline/writingPhase.ts`：
- 集成周期性检查

### Phase 5: 自主推进与结构规划

`server/src/services/pipeline/autoScheduler.ts`（新建）：
- 自动续写调度器
- 全书完成度检测

`server/src/services/pipeline/writingPhase.ts`：
- 自动续写集成
- 偏离度三维度检查（情节/人设/世界观）

### Phase 6: 可观测性与用户体验

`server/src/routes/novels.ts`：
- 新增 GET /api/novels/:id/health 健康仪表盘 API

`client/src/components/workspace/HealthDashboard.tsx`（新建）：
- 全书健康概览（章数、字数、进度、质量）
- 质量指标（AI味、字数合规、风格、信息密度）
- 生命周期（钩子/伏笔回收率、主线完成度）
- 角色健康（活跃/沉默角色）
- 情绪曲线图
- 质量趋势图

`client/src/styles/components.css`：
- HealthDashboard 组件样式

## 验证

- TypeScript 编译：shared + server + client 均通过
- 质量后验：每章有 qualityScore 和 ChapterQualityLog
- 续写模式：15 条规则 + 状态追踪 + 记忆管理
- 周期检查：每 10 章自动运行
- 自动续写：autoContinue=true 时自动调度
- 健康仪表盘：API 返回完整指标

## 风险说明

- **LLM 调用增量**：每章增加约 600 output tokens（质量评分 300 + 章纲预检查 300），成本增加约 10%
- **自动续写调度**：使用 setTimeout 延迟 5 秒，进程重启后不会自动恢复
- **角色漂移检测**：每 10 章一次 LLM 调用，可选功能
- **数据库 Schema 变更**：需要执行 `pnpm prisma:push` 同步到数据库
