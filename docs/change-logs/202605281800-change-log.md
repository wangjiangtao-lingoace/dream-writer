# 变更记录

## 变更原因
用户反馈 AI 分析进度提示藏在面板内部（左下角），在写作页完全看不到。各种提示（中断、失败等）不够明显。

## 修改点

### 全局 AI 浮动进度条
- `client/src/pages/NovelWorkspace.tsx`
  - 新增 `aiProgress` 状态和 `pipelinePollRef` 轮询引用
  - 新增 `useEffect`：每 5 秒轮询 `/api/pipeline/novel/:id` 检测流水线状态
  - 流水线运行时在工作台顶部显示醒目的浮动进度条（蓝色渐变背景 + spinner + 消息 + 进度百分比 + "查看详情"按钮）
  - 进度条在写作布局和标准布局中都显示

### Notice-bar 样式增强
- `client/src/pages/NovelWorkspace.tsx`
  - notice-bar 增加 `fontWeight: 500` 提升可读性
  - 错误类型 notice（含"失败"/"错误"）使用红色背景 `var(--error-muted)` + 红色文字 `var(--error)`

## 风险说明
- 轮询间隔 5 秒，对后端压力可忽略
- 流水线不存在时 API 返回 404，静默处理不影响用户体验
- 进度条为纯前端展示，不改变任何业务逻辑

## 验证情况
- TypeScript 编译通过
