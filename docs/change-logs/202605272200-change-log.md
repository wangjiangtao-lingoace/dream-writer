## 变更原因

实现阶段二 Task 6：RAG 管理 API 和数据回填脚本。

## 修改点

### 新建文件
- `server/src/routes/rag.ts` — RAG 管理 API（4 个端点）
- `server/scripts/backfill-rag.ts` — 数据回填脚本

### 修改文件
- `server/src/app.ts` — 注册 `/api/rag` 路由

### 功能变更

1. **RAG 管理 API**
   - `GET /api/rag/status` — 索引统计（向量可用性、模型信息、chunk 数量、按类型/作品分布）
   - `POST /api/rag/reindex/:novelId` — 重建指定作品索引
   - `POST /api/rag/reindex-global` — 重建全局索引
   - `POST /api/rag/search` — 测试搜索接口
   - 所有端点检查 ENABLE_RAG，未启用返回 503

2. **回填脚本**
   - 自动加载 .env，强制 ENABLE_RAG=true
   - 三阶段遍历：KnowledgeAsset → Memory → GeneralKnowledge
   - 逐条调用 ingestText()，实时进度输出
   - 支持 `--dry-run` 参数
   - 失败数 > 0 时退出码 1

## 使用说明

```bash
# 查看 RAG 索引状态
curl http://localhost:3000/api/rag/status

# 重建指定作品索引
curl -X POST http://localhost:3000/api/rag/reindex/{novelId}

# 重建全局索引
curl -X POST http://localhost:3000/api/rag/reindex-global

# 测试搜索
curl -X POST http://localhost:3000/api/rag/search \
  -H "Content-Type: application/json" \
  -d '{"query": "修仙", "novelId": "xxx"}'

# 运行回填脚本
cd server && npx ts-node scripts/backfill-rag.ts
cd server && npx ts-node scripts/backfill-rag.ts --dry-run
```
