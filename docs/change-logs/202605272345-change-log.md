# 变更记录

## 变更原因
UI/UX 现代化第二轮：修复 CharacterCard、KnowledgeHub、MemoryPanel 三个组件共 44 个 CSS 类名缺少定义的问题。这些组件仅依赖内联样式，结构性布局类完全没有 CSS 定义。

## 修改点
1. **components.css**: 新增 44 个 CSS 类定义，覆盖共享面板/表单/卡片类、CharacterCard、KnowledgeHub、MemoryPanel 四个模块
2. **CharacterCard.tsx**: 修复 1 处硬编码琥珀色 `rgba(139,69,19,0.08)` → `var(--accent-muted)`
3. **KnowledgeHub.tsx**: 修复 2 处硬编码琥珀色 `rgba(139,69,19,0.08)` → `var(--accent-muted)`
4. **MemoryPanel.tsx**: 剧情状态面板从硬编码亮色 (`#f8fafc`/`white`/`#6b7280`) 改为 CSS 类 + 暗色主题变量；记忆层级颜色从 pastel 亮色改为语义色变量

## 文件列表
- `client/src/styles/components.css` — 新增 ~350 行 CSS 定义
- `client/src/components/CharacterCard.tsx` — 修复 1 处硬编码色值
- `client/src/components/KnowledgeHub.tsx` — 修复 2 处硬编码色值
- `client/src/components/MemoryPanel.tsx` — 修复剧情状态面板和层级颜色

## 风险说明
- 纯视觉层改动，不影响接口或业务逻辑
- 部分组件仍有内联样式覆盖 CSS 类（优先级高于类定义），后续轮次统一清理
- CharacterCard 的 `roleColorMap` 仍使用硬编码 hex 色值（功能性正常，暂不处理）

## 验证情况
- TypeScript 类型检查通过
- Vite dev server 已验证组件样式正确应用
