# 变更记录：Pipeline 流程优化 — 大纲先行、分步确认

## 变更原因
独立创作模式下 Pipeline 一次性生成全部内容（大纲→世界观→人物→风格→卷纲→章纲→钩子→章节），浪费 Token 且内容不可控。用户需求：先生成大纲供审核确认，确认后再生成其余资产。

## 修改点

### 新流程（standalone 模式）
```
Phase 1: outline     → 生成大纲 → 暂停，用户查看/编辑/重新生成 → 确认
Phase 2: generation  → 世界观+人物+风格+卷纲+章纲+钩子 → 暂停，用户查看/编辑/重新生成 → 确认
Phase 3: writing     → 前 1-3 章草稿 → 完成
```
仿写模式（imitation）保持原有 3 阶段流程不变。

### 具体改动
1. PipelineConfig 增加 `mode: "standalone" | "imitation"` 字段
2. executePipeline 根据 mode 分支：standalone 走大纲先行流程，imitation 走原有流程
3. 新增 executeOutlinePhase（只生成大纲）和 executeGenerationPhase（世界观+人物+风格+卷纲+章纲+钩子）
4. confirmPhase 支持新阶段流转：outline → generation → writing
5. resumePipeline 支持新阶段恢复
6. regenerateStep 支持 outline/generation 阶段的所有步骤
7. userHint 注入 LLM prompt（修复原有 bug：userHint 只存记录未传 prompt）
8. PipelinePage 增加 outline/generation 阶段标签，阶段列表动态化
9. PipelinePage 成果查看区域接入步骤级交互：展开完整内容、编辑、重新生成（带 hint）、确认（带 feedback）

## 风险说明
- 仿写模式流程完全不变，不受影响
- standalone 模式下大纲阶段只消耗约 2000 token，确认后才生成后续内容
- userHint 注入 prompt 后，重新生成会参考用户意见
