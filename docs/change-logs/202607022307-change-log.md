# 变更日志 2026-07-02 23:07

## 变更原因
基于系统三维评审报告（使用用户/编辑/读者），针对5个核心改进项进行优化，提升剧情系统的自动化程度和内容质量。

## 修改点

### 1. 主线里程碑自动追踪（P0）
- 新建 `MainlineService.ts`，提供里程碑覆盖检测
- 章节写完后自动校验主线里程碑是否被覆盖
- 使用关键词匹配（40%阈值）+ 角色名匹配
- Mainline 模型新增 `milestoneStatus` 字段记录完成状态

### 2. 正文级别钩子/伏笔回收自动确认（P0）
- 在 postProcessing.ts 中添加 `detectResolutionsInContent()` 函数
- 使用滑动窗口模糊匹配（阈值 0.7）检测正文中的回收证据
- 命中后立即更新 Hook/Foreshadow 状态，不依赖 LLM
- ChapterSummary 新增 `resolvedForeshadows` 字段

### 3. 角色成长检查点机制（P1）
- periodicCheck.ts 新增 `checkCharacterGrowthCheckpoints()` 函数
- 超过 10 章无成长记录告 warning，超过 15 章告 critical
- postProcessing.ts 集成成长检查点写入（LLM 返回 growthCheckpoint 时）
- Character 模型新增 `growthCheckpoints` 字段

### 4. StoryState 事件驱动自动流转（P1）
- StoryStateService.ts 新增规则引擎 `TRANSITION_RULES`
- 3条规则：setup→development（前3章）、development→climax（压抑>7）、climax→resolution（伏笔回收）
- 在 postProcessing.ts 中非阻塞调用，不影响现有流程

### 5. AI 味深度优化（P2）
- aiSmellWords.ts 新增 3 个深度检测函数：
  - `detectSentencePatterns()` - 句式模式（6维度）
  - `detectParagraphPatterns()` - 段落模式（5维度）
  - `detectAIStructure()` - 结构模式（5维度）
- 综合评分：词汇 40% + 句式 30% + 段落 20% + 结构 10%

## 风险说明
- 主线里程碑检测使用关键词匹配，可能存在误判（简化方案）
- 钩子/伏笔回收检测阈值 0.7，对中文场景足够宽松
- StoryState 自动流转是"建议"机制，不覆盖手动设置
- AI 味深度检测为纯正则/统计，无 LLM 调用，性能无风险
- Schema 变更需要执行 `pnpm prisma:push` 或手动 ALTER TABLE
