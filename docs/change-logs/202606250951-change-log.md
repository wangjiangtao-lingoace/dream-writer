# Pipeline canonical 续写章纲偏移修复

## 变更原因

当前创作中的《人在阳间打工，老祖阴间享福》已有用户原文 1-3 章，并已标记为 canonical，但流水线生成章纲时仍从第 1 章重新规划，导致后续第 4 章可能使用错误章纲承接。

同时，前端确认不存在的阶段结果时，后端直接暴露 Prisma 更新失败，形成 500 错误，不利于排查。

## 修改点

1. `chapterOutlinesPhase` 增加 canonical 起点计算：
   - 查找最后一个 `user_original / isCanonical / canRewrite=false` 章节
   - 后续章纲从 `lastCanonical + 1` 开始
   - 章纲 prompt 明确提示前 N 章为用户原文，只能承接，不得重写

2. `persistVolumeChapterData` 支持 canonical offset：
   - 按真实全书章节号写入 `ChapterOutline.sortOrder`
   - 遇到 canonical 正文章节号时跳过章纲写入，避免覆盖用户原文位置

3. `confirmPhase` 增加缺失阶段结果容错：
   - Prisma `P2025` 转换为明确业务错误：`阶段结果不存在：phase/step`

4. 增加回归测试：
   - canonical 后章纲起点计算
   - 全书章节范围提示文案
   - confirm 缺失阶段结果错误处理

## 注意事项

本次修复只修代码，不直接改当前数据库。当前数据库中已经生成的错误章纲仍需清理或重新生成后续章纲，建议先备份 `server/dev.db`，再删除新作品的旧 `ChapterOutline/Hook/Foreshadow/PleasurePoint/EmotionCurve` 规划产物并恢复流水线。
