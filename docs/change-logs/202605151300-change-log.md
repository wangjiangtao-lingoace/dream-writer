# 变更记录 2026-05-15 13:00

## 变更原因

实现阶段一 Task 4：文件导入支持，支持用户上传 PDF/EPUB/TXT 文件作为拆书素材。

## 修改点

### 新建文件
- `server/src/services/TextExtractionService.ts` — 文本提取服务

### 修改文件
- `server/src/routes/upload.ts` — 添加文档上传路由
- `client/src/pages/AnalyzeCreate.tsx` — 添加文件上传按钮
- `server/package.json` — 添加 pdf-parse 和 epub2 依赖

### 功能变更

1. **TextExtractionService（文本提取服务）**
   - `.txt`：去除 BOM，规范化换行
   - `.pdf`：使用 `pdf-parse` v2 提取文本
   - `.epub`：使用 `epub2` 遍历 spine 章节，去除 HTML 标签
   - 所有输出截断到 12000 字符
   - EPUB 章节空值防护

2. **upload.ts 路由增强**
   - 新增 `documentStorage` multer 配置，写入 `uploads/documents/`
   - 新增 `POST /api/upload/document` 路由
   - MIME 类型过滤（PDF/EPUB/TXT），5MB 限制
   - 文件类型拒绝时返回明确错误信息
   - 路径穿越漏洞修复（`path.basename()`）

3. **AnalyzeCreate.tsx 前端集成**
   - 隐藏文件输入，accept=".txt,.pdf,.epub"
   - "导入文件"按钮触发文件选择
   - 使用 `fetch` + `FormData` 上传（因为 api 工具仅支持 JSON）
   - 返回文本自动填充 `pastedText`
   - 标题为空时自动用文件名填充
   - 加载状态指示器

- EPUB 图片章节会被跳过
- 临时文件存储在 `uploads/documents/`，提取后立即删除
