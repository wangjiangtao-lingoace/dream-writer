# 变更记录：灵感编辑功能 + Prompt 增量补充优化

## 变更原因
1. 用户在生成大纲后常需修改灵感描述，但 NovelWorkspace 中灵感为只读展示
2. 用户灵感素材通常已经非常详细，LLM 重新生成大纲反而导致内容降级（丢失细节、压缩表达）
3. 需要改为增量补充模式：保留原文 + 只补充缺失部分

## 修改点

### 一、灵感编辑功能
- NovelWorkspace 新增 `editingInspiration`/`inspirationDraft` 状态
- 灵感展示区域改为可编辑模式：编辑按钮 → textarea → 保存/取消
- 保存时调用 `PUT /api/novels/:id` 更新灵感，成功后刷新本地状态

### 二、Prompt 增量补充模式（4 个核心方法）

| 方法 | 改进 |
|------|------|
| `generateOutline` | 改为增量补充模式：保留灵感原文 + 只补充缺失的结构化字段 |
| `generateWorldview` | 改为增量补充模式：保留大纲中的世界观设定 + 只补充缺失细节 |
| `generateCharacters` | 改为增量补充模式：保留大纲中的人物设定 + 只补充外貌/能力等缺失字段 |
| `generateStyle` | 改为增量补充模式：保留大纲中的风格描述 + 只补充缺失的风格参数 |

核心原则：
- 用户创意是核心素材，必须最大程度保留原文内容和表达
- 只补充用户未涉及的部分，不改写已有内容
- 如果素材已经非常完整，补充量应尽可能少

### 三、其他 Prompt 优化
| 方法 | 改进 |
|------|------|
| `generateVolumeOutline` | 注入世界观+人物+风格 + 修复"新Chars2"拼写错误 |
| `generateChapterOutlines` | 注入大纲+世界观+人物+风格 |
| `generateMainlinesAndHooks` | 注入完整大纲+世界观+人物 + 新增 userHint 参数 |

### 四、上下文传递链优化
- `executeGenerationPhase` 中每个步骤接收前序步骤的完整输出
- `executeStructuringPhase` 中卷纲/章纲/钩子生成也注入世界观+人物+风格
- `regenerateStep` 支持 volume/chapter_outline/mainline_hook 步骤的重新生成
