# 变更记录：卷纲菜单式 UI

## 变更原因
1. 原卷纲展示为扁平列表，编辑/删除按钮样式不明显
2. 缺少卷纲与章纲的层级导航
3. 无法在卷纲页面直接生成/查看章纲
4. 需要菜单式交互：卷纲列表 → 卷详情 + 章纲列表 → 章纲详情

## 修改点

### 一、PipelinePage.tsx

#### 1.1 新增状态变量
- `selectedVolume: number | null` — 当前选中的卷索引，null 表示卷列表视图
- `expandedChapters: Set<number>` — 展开的章纲索引集合

#### 1.2 新增按钮样式
- `btnPrimaryStyle` — 主按钮样式（accent 背景色）
- `btnDangerStyle` — 危险按钮样式（红色边框）

#### 1.3 新增数据解析函数
- `getVolumeOutlineData()` — 解析卷纲 JSON 数据
- `getChapterOutlineData(volIndex)` — 解析指定卷的章纲 JSON 数据
- `getChapterOutlineStatus(volIndex)` — 获取章纲确认状态
- `getVolumeOutlineStatus()` — 获取卷纲确认状态

#### 1.4 新增 `renderPlanningSection()` 方法
实现卷纲菜单式 UI，包含两种视图：

**卷列表视图**：
- 卷卡片列表，每卷显示：序号、标题、目标摘要、章纲数量/状态
- 点击进入卷详情视图

**卷详情视图**：
- 面包屑导航（返回卷纲列表）
- 卷纲详情卡片（目标、关键事件、转折点、高潮）
- 操作按钮：编辑、重新生成卷纲、确认卷纲
- 章纲列表，每章可展开查看详情
- 章纲详情包含：目标、冲突、情绪基调、章末钩子、出场角色、埋设/回收钩子、埋设/回收伏笔、爽点
- 章纲标签：高潮（金色）、转折（紫色）
- 操作按钮：重新生成章纲、确认本卷章纲
- 空章纲状态：显示"生成本卷章纲"按钮

#### 1.5 更新结果展示区域
- planning 阶段结果使用专用菜单式 UI
- 其他阶段结果保持原有扁平列表
- 新增编辑/重新生成/确认面板（支持 planning/ 前缀的 key）

### 二、样式说明
- 所有按钮使用 `btnSmallStyle`、`btnPrimaryStyle`、`btnDangerStyle` 统一样式
- 按钮带边框、圆角、悬停效果，符合按钮状态
- 层级导航使用面包屑模式
- 章纲展开/收起使用箭头旋转动画
