# 变更记录 2026-05-27 20:00

## 变更原因

实现阶段二 Task 4：混合检索服务（RagRetrieveService），实现向量 + BM25 + RRF 融合检索。

## 修改点

### 新建文件
- `server/src/services/RagRetrieveService.ts` — 混合检索服务

### 修改文件
- `server/src/services/RagIngestService.ts` — 添加 FTS5 同步钩子
- `server/src/app.ts` — 添加优雅关闭处理

### 功能变更

1. **RagRetrieveService 核心功能**
   - FTS5 虚表 `rag_chunks_fts`（unicode61 tokenizer）用于 BM25 检索
   - 向量检索：调用 EmbeddingService + VectorStore
   - RRF 融合：`score = Σ weight / (k + rank + 1)`，k=60
   - `retrieve(query, { novelId, topK?, vectorWeight?, bm25Weight? })` 返回格式化上下文
   - 降级策略：嵌入不可用 → 仅 BM25；sqlite-vec 不可用 → 仅 BM25；索引为空 → 空字符串

2. **FTS5 安全修复**
   - `escapeFts5Query()`: 转义 FTS5 特殊字符，防止查询注入
   - `novelId` 空字符串校验，防止 BM25/向量检索范围不一致

3. **FTS5 同步钩子**
   - `syncFtsAfterInsert(chunks)`: 写入 RagChunk 后同步到 FTS5
   - `syncFtsAfterDelete(ownerType, ownerId)`: 删除 RagChunk 后清理 FTS5
   - RagIngestService 在 createMany/rollback/deleteChunks 后调用

4. **优雅关闭**
   - app.ts 添加 SIGTERM/SIGINT 处理
   - 关闭 RagRetrieveService、VectorStore、Prisma 连接

## 风险说明

- FTS5 unicode61 tokenizer 对中文分词支持有限，可能影响 BM25 精度
- 三个独立的 better-sqlite3 连接（Prisma、VectorStore、FTS5），依赖 WAL 模式保证一致性
- FTS5 与 Prisma 数据不同步时需手动重建索引
