# 202607022305 - 角色成长检查点机制

## 变更原因
实现角色成长检查点（Growth Checkpoints）机制，用于追踪每个角色在各章节的成长事件，并在周期性一致性检查中检测角色弧线停滞问题。

## 修改点

### 1. Character 模型新增字段
- 新增 `growthCheckpoints String?` 字段，存储 JSON 数组：`[{chapter, description, type, createdAt}]`
- 记录角色在哪些章节发生了成长事件（能力提升/性格转变/关系变化等）

### 2. 周期性检查新增成长检查点检测
- 新增 `GrowthCheckpointAlert` 接口
- 新增 `checkCharacterGrowthCheckpoints()` 函数：检测活跃角色是否超过 10/15 章无成长记录
- 集成到 `runPeriodicConsistencyCheck()` 主流程
- 告警结果持久化到 `ConsistencyIssue` 表（type: `character_growth`）

### 3. 后处理流程自动记录成长检查点
- LLM prompt 中 `characterUpdates` 新增 `growthCheckpoint` 字段
- 当 LLM 判定角色本章有成长时，自动写入 `growthCheckpoints` JSON 数组
- 包含同章去重逻辑，上限 50 条自动裁剪

## 风险说明
- `growthCheckpoints` 为可选字段，不影响现有数据
- `checkCharacterGrowthCheckpoints()` 仅查询有 arcSummary/arcDetail 的活跃角色，不影响无成长线设定的角色
- prisma db push 因 `rag_vectors` 虚拟表报错，已通过直接 ALTER TABLE 解决，后续需关注 Prisma 兼容性
