# 变更记录 202606121500

## 变更原因

基于 `docs/OPTIMIZATION_REPORT.md` 和 `docs/OPTIMIZATION_PLAN.md` 的优化分析结果，修复 P0/P1 级问题，将项目从 Alpha（60分）升级到 Beta（70分）阶段。

## 修改点

### P0-1: 修复 CharacterRelation TypeScript 类型错误（17 处）

Prisma Schema 中 `CharacterRelation` 模型定义了 `charA`/`charB` 关系字段和 `charAId`/`charBId` 标量字段，但代码中将 `charA`/`charB` 当作字符串（角色名）使用，导致类型不匹配。

- 所有查询改为使用 Prisma 关系过滤器：`{ charA: { name: xxx } }`
- 查询结果通过 `include` 加载关系数据
- `CharacterImportService` 改为先查角色 ID，再创建关系
- 同步迁移数据库 Schema（charA/charB TEXT → charAId/charBId FK）

### P0-2: 移除硬编码 API Key

`server/.env.example` 中包含真实 MIMO API Key，存在安全泄露风险。已替换为占位符并添加安全警告。

### P0-3: 同步文档状态

CLAUDE.md 路线图状态与 README.md 不一致（P1 显示"待启动"，实际 P0-P3 已完成）。已同步更新。

### P1-1: 完善错误处理

- 新增 `server/src/middleware/errorHandler.ts`：统一错误处理中间件

  - 未知错误兜底
- 更新 `server/src/app.ts`：替换原有错误处理为统一中间件

## 风险说明

- **数据库迁移**：CharacterRelation 表从 charA/charB（TEXT）迁移到 charAId/charBId（FK），旧数据通过 JOIN Character 表迁移。当前表为空，无数据丢失风险。
- **错误处理变更**：原有的 ZodError 处理逻辑已合并到新的 errorHandler 中间件中，行为一致。
- **API Key 泄露**：.env.example 中的 Key 已替换，但 Git 历史中仍存在。建议后续使用 git filter-branch 清理。
- **测试文件排除**：tsconfig.json 排除了 `__tests__` 目录，测试文件仅通过 Jest 运行，不影响构建产物。
