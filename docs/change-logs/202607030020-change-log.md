# 202607030020 - P0 安全漏洞修复

## 变更原因
修复两个 P0 级安全漏洞：路径遍历漏洞和缺少 API 认证中间件。

## 修改点
1. **路径遍历漏洞修复**：`DELETE /api/upload/file` 接口原先基于 `process.cwd()` 拼接路径，攻击者可通过 `../../etc/passwd` 等路径逃逸到任意目录删除文件。改为基于 `uploadDir` 拼接并校验路径前缀。
2. **API Key 认证中间件**：新增 `apiKeyAuth` 中间件，从环境变量 `API_SECRET_KEY` 读取密钥。未配置时跳过认证（开发模式），配置后要求请求携带 `x-api-key` header 或 `apiKey` query 参数。

## 文件列表
| 文件 | 变更说明 |
|------|---------|
| `server/src/routes/upload.ts` | 修复删除文件接口的路径遍历漏洞：`path.resolve(process.cwd(), ...)` 改为 `path.resolve(uploadDir, ...)`，增加 `startsWith` 路径逃逸校验，失败返回 403 |
| `server/src/middleware/auth.ts` | 新建 API Key 认证中间件，支持 header/query 两种方式传递 key，未配置 `API_SECRET_KEY` 时跳过认证 |
| `server/src/app.ts` | 导入并注册 `apiKeyAuth` 中间件，挂载于 `/api` 路径下、所有业务路由之前 |

## 风险说明
- `apiKeyAuth` 中间件影响所有 `/api` 下的请求。未配置 `API_SECRET_KEY` 时自动跳过，不影响现有开发流程。
- 路径遍历修复后，删除文件必须位于 `uploadDir` 目录内。前端传入的 `path` 参数需为相对于 `uploadDir` 的路径（与上传时返回的相对路径一致则无影响）。

## 验证情况
- `pnpm typecheck` (server) 通过，无类型错误。
