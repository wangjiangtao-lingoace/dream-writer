# 变更记录

## 变更原因
自主创作创建小说后，AI 分析不会自动启动，用户需要手动切换到"创作总控"面板才能触发。用户体验断裂。

## 修改点

### NovelForm 自动启动 Pipeline
- `client/src/pages/NovelForm.tsx`
  - 创建小说成功后，自动调用 `POST /api/pipeline/start` 启动 standalone 模式的 AI 流水线
  - Toast 提示从"作品创建成功"改为"作品创建成功，AI 正在分析..."
  - 移除 `?new=true` 参数（不再需要，默认已是写作页）
  - Pipeline 启动失败不阻塞跳转，用户可从创作总控手动启动

## 调用链路
```
NovelForm.handleSubmit()
  → POST /api/novels (创建小说)
  → POST /api/pipeline/start (自动启动 AI 流水线)
  → navigate(/novel/:id) (跳转到工作台)
  → 工作台轮询 pipeline 状态，顶部显示进度条
```

## 当前 AI 模型
- Provider: DeepSeek
- Model: deepseek-chat
- 配置来源: `.env` 环境变量 (`DEFAULT_LLM_PROVIDER=deepseek`)

## 风险说明
- Pipeline 启动为 fire-and-forget，失败不影响用户进入工作台
- 用户仍可从"创作总控"面板手动启动或管理 pipeline
