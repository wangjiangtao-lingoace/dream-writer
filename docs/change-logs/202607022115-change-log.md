# 变更日志 2026-07-02 21:15

## 变更原因
前端 PipelinePage 使用 setInterval 每3秒轮询 REST API 获取 pipeline 进度，后端已有 SSE 端点 `GET /api/pipeline/:jobId/stream` 但未被前端消费。本次变更用 SSE 替换轮询，实现实时推送。

## 修改点
1. 新建 `usePipelineSSE` hook 封装 SSE 连接逻辑
2. PipelinePage 接入 SSE，替换原有轮询机制
3. 添加 SSE 失败时自动回退到轮询的容错机制

## 文件列表

### 新增文件
- `client/src/hooks/usePipelineSSE.ts` — SSE 连接 hook，使用 fetch + ReadableStream 模式，支持自动重连（3次指数退避）和回退

### 修改文件
- `client/src/pages/PipelinePage.tsx` — 接入 SSE hook，添加 jobId/useFallbackPolling 状态，修改 loadPipeline 提取 jobId，handleResume 后重建 SSE 连接

## 风险说明
- SSE 连接失败时自动回退到轮询，确保功能可用性
- 防抖 200ms 避免 SSE 突发事件导致重复 REST 请求
- 组件卸载或 pipeline 终止时自动清理连接
