# 变更记录：DashboardPanel 内联样式迁移至 CSS 类

## 变更原因
DashboardPanel.tsx 中有约 50+ 个内联 `style={{}}` 对象，维护困难且不一致。将高频/重要样式提取为 CSS 类，提升可维护性。

## 修改点
- 新建 `dashboard.css`，定义布局容器、卡片、通知栏、动作按钮、流程节点、进度条、采用状态、结果展示等 40+ 个 CSS 类
- DashboardPanel.tsx 内联样式从 ~50+ 个减少到 5 个（保留 SVG 尺寸、动态宽度、单次使用的 wrapper）
- 使用 BEM 风格修饰符处理条件样式（`--done`、`--continue`、`--accent`）
- 进度条 `width` 百分比保持内联（动态值）

## 文件列表
- `client/src/styles/components/dashboard.css`（新建）— 所有 dashboard 相关 CSS 类
- `client/src/components/workspace/DashboardPanel.tsx`（修改）— 替换内联样式为 CSS 类名引用
- `docs/change-logs/202605291125-change-log.md`（新建）— 变更记录

## 风险说明
- 视觉输出应与迁移前完全一致，CSS 属性值与原内联样式一一对应
- 唯一的行为差异：`dashboard-action-btn--continue:disabled` 现在会显式恢复 `border-color` 和 `cursor`，与原内联逻辑一致
- 不影响任何 TypeScript 逻辑、状态管理或事件处理

## 验证情况
- `npx tsc --noEmit --project client/tsconfig.json` 通过，无类型错误
- 剩余 5 个内联样式均为合理保留（动态宽度、SVG 尺寸、单次使用 wrapper）
