# 变更记录 - 2026-05-14 03:00

## 变更原因
正文生成路由 `/api/ai/chapter-content/:novelId/:chapterId` 使用"模拟流式"——先 await LLM 返回完整内容，再按 50 字符分块以 20ms 间隔发送给前端。这导致用户必须等待整个生成过程完成后才能看到第一个字，体验差且浪费等待时间。

## 修改点
1. `LlmInvokeService` 新增通用 `streamText()` 方法，接受 `{ system, prompt, temperature, maxTokens }` 参数，通过 OpenAI 兼容 SSE 流式 API 返回 `AsyncGenerator<string>`。
2. `AIService` 新增 `generateChapterContentStream()` 异步生成器函数，构建与原 `generateChapterContent()` 相同的丰富 prompt（含世界观、人物、章纲、记忆、剧情状态等），但调用 `streamText()` 而非 `completeText()`。
3. 路由层改为 `for await (const chunk of generateChapterContentStream(...))` 直接消费流，每收到一个 token 即刻推送给前端。

## 文件列表
| 文件 | 变更说明 |
|------|----------|
| `server/src/services/llm/LlmInvokeService.ts` | 新增 `streamText()` 通用流式方法和 `streamFallbackRaw()` 兜底方法 |
| `server/src/services/AIService.ts` | 新增 `generateChapterContentStream()` 异步生成器，复用原有 prompt 构建逻辑 |
| `server/src/routes/ai.ts` | 正文生成路由改为直接消费流式生成器，移除模拟分块逻辑 |

## 风险说明
- **低风险**：原有 `generateChapterContent()` 函数保持不变，非流式调用方不受影响。
- `streamText()` 复用了 `streamChapterDraft()` 相同的 fetch + SSE delta 解析逻辑，经过已有验证。
- 若 LLM API 不可用，`streamFallbackRaw()` 会返回错误提示而非静默失败。
- `res.writableEnded` 检查防止客户端断开后继续写入。
