# EmbeddingService 代码质量修复 (I1-I5)

## 变更原因
代码质量审查发现 EmbeddingService 存在 5 个 Important 级别问题，涉及安全漏洞、数据校验缺失和缓存逻辑缺陷。

## 修改点
1. **I1 chunkOverlap 校验**: `chunkText` 入口处将 overlap 钳制为 `min(overlap, chunkSize-1)`，防止 `text.slice(0)` 或负偏移
2. **I2 SSRF 防护**: `callEmbeddingApi` 中校验 URL 协议，仅允许 `http:`/`https:`，拒绝 `file:`/`ftp:` 等
3. **I3 返回数量校验**: API 返回的 `data` 数组长度必须与输入 `texts` 长度一致，否则抛出异常
4. **I4 null 缓存修复**: `getConfig()` 在 `resolveConfig()` 返回 null 时清空 `configPromise`，下次调用重试
5. **I5 embedding 数值校验**: 验证每个 embedding 为非空数组、维度匹配、且不含 NaN/Infinity

## 文件列表
- `server/src/services/EmbeddingService.ts` — 上述 5 项修复

## 风险说明
- I2 的协议校验在 `new URL()` 构造时完成，非法 URL 会直接抛异常，无额外副作用
- I4 的 null 缓存清除可能导致首次配置未就绪时多次查询 DB，但频率极低且可接受
- I5 的维度校验依赖 `config.dimension` 正确性，若配置错误会在首次调用时报错（符合预期）
