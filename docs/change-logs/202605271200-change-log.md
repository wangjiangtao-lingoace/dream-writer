# 变更记录 2026-05-27 12:00

## 变更原因

实现阶段二 Task 1：向量存储层（sqlite-vec 集成），为 RAG 检索提供向量存储能力。

## 修改点

### 新建文件
- `server/src/db/vectorStore.ts` — 向量存储层，集成 sqlite-vec 扩展

### 功能变更

1. **VectorStore 类**
   - 使用独立的 better-sqlite3 连接（与 Prisma 共用 dev.db）
   - 加载 sqlite-vec 扩展，创建 vec0 虚表
   - 维度验证：整数范围 [1, 65536]，防止 SQL 注入
   - 优雅降级：sqlite-vec 加载失败时 `available = false`，所有操作静默跳过

2. **vec0 虚表 schema**
   - `chunk_id TEXT PRIMARY KEY` — 切片唯一标识
   - `owner_type TEXT` — 所属类型（knowledge_asset / memory / general_knowledge）
   - `owner_id TEXT` — 所属记录 ID
   - `novel_id TEXT` — 作品 ID
   - `embed_model TEXT` — 嵌入模型名称
   - `embedding float[N]` — 向量数据

3. **公共 API**
   - `insertVectors()` — 批量插入/更新向量（事务）
   - `search()` — KNN 向量检索，支持 novelId/ownerTypes 过滤和 distance 阈值
   - `deleteByOwner()` — 按 owner 删除向量
   - `deleteByNovel()` — 按作品删除向量
   - `getStats()` — 返回索引统计信息
   - `isAvailable()` — 检查 sqlite-vec 是否可用
   - `close()` — 关闭连接，清理单例

4. **安全修复**
   - C1：dimension 整数范围验证，防止 SQL 注入
   - C2：loadSqliteVec + ensureTable 统一 try-catch 保护
   - I2：close() 设置 available = false 并清理单例引用

## 风险说明

- sqlite-vec 二进制可能与特定 Node 版本不兼容，已做降级处理

- 向量维度在构造时固定，变更嵌入模型需重建索引
