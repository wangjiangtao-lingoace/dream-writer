# 变更记录

## 变更原因
优化流水线信息库注入方式和前端反馈，提升 LLM 对关键信息的关注度，改善用户体验。

## 修改点

### 1. contextBuilders.ts - 优化注入顺序
- 将核心卖点（coreSellingPoint）和核心矛盾（coreConflict）移至 context 最前面
- 将主角信息（role="主角"）优先于其他角色展示
- 新顺序：核心设定 → 主角 → 作品大纲 → 其他角色 → 世界观 → 卷纲/章纲 → 主线 → 钩子 → 记忆 → 知识库

### 2. layer4-characters.ts - 限制角色数量
- 角色约束 prompt 最多显示 8 个角色
- 超过 8 个时按优先级排序：主角 > 反派 > 配角 > 其他
- 在 prompt 末尾添加提示："（共N个角色，仅显示最重要的8个）"

### 3. layer6-chapter-task.ts - 优化 null outline 处理
- 将默认提示从严格约束改为更合理的自然延续指引
- 新提示："本章没有章纲数据。请根据前文自然延续剧情，保持角色一致性，推进主线发展。"

### 4. PipelinePage.tsx - 添加信息库引用面板
- 在接口中添加 metadata 字段支持
- 添加引用信息折叠面板 UI
- 显示格式：标签形式，如 "角色:张三" "世界观:修仙体系"
- 支持 assetType 中文翻译映射

## 文件列表
- `server/src/services/pipeline/contextBuilders.ts` - 调整资产注入顺序
- `server/src/services/pipeline/prompts/layer4-characters.ts` - 添加角色数量限制
- `server/src/services/pipeline/prompts/layer6-chapter-task.ts` - 优化空章纲提示
- `client/src/pages/PipelinePage.tsx` - 添加引用信息面板 UI

## 风险说明
- contextBuilders.ts 的顺序调整可能影响 LLM 输出风格，需观察生成质量
- 角色限制（8个）可能遗漏次要角色，但优先保留了核心角色
- metadata 字段当前为占位 UI，需后端后续填充数据
