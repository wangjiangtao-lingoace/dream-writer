# 变更记录

## 变更原因
1. 续写模式（ContinuationService）是独立服务，无全局规划、无用户干预，与自主创作/拆书仿写的 Pipeline 流程不统一。需改造为走 Pipeline 流水线，先分析已有内容提取资产，再规划卷纲/章纲，每个阶段用户可确认/修改。
2. 全面中文化：消除所有残留英文（如 DashboardPanel 中的 "Pipeline"），确保全部用户可见文本为中文。

## 修改点

### 一、续写模式改造

#### 后端 PipelineConfig 扩展
- `server/src/services/PipelineService.ts`
  - `mode` 联合类型新增 `"continue"`
  - `startPipeline` 中 totalSteps 计算增加 continue 分支
  - `executePipeline` 新增 `mode === "continue"` 分支，调用 `executeAnalyzePhase_continue`

#### 新增续写分析阶段
- `server/src/services/pipeline/analyzePhase.ts`
  - 新增 `executeAnalyzePhase_continue()` 函数
  - 从已有章节内容提取资产（代替灵感文本）
  - 三步：analyze（分析已有内容）→ decompose（拆解入库）→ outline（从章节推断大纲）
  - 新增 `generateOutlineFromChapters()` 专用大纲生成函数

#### 提取共享后处理工具
- `server/src/services/pipeline/postProcessing.ts`（新建）
  - 从 ContinuationService 提取 `updateStoryState()` 和 `extractMemories()`
  - ContinuationService 和 writingPhase 都引用此模块

#### writingPhase 后处理增强
- `server/src/services/pipeline/writingPhase.ts`
  - continue 模式下，正文生成后执行：更新 StoryState + 提取记忆

#### workflow-status 识别续写模式
- `server/src/routes/novels.ts`
  - 新增 `isContinue` 判断：有已写章节 + 无拆书分析 + 无仿写方案
  - `creationMode` 返回 `"continue"`
  - nextActions 新增"智能续写"按钮
  - adoption 值 "已被 Pipeline 使用" → "已被流水线使用"（4 处）

#### ContinuationService 废弃标记
- `server/src/services/ContinuationService.ts`
  - 移除本地 `updateStoryState` 和 `extractMemories` 函数（改为从 postProcessing 导入）

### 二、前端改动

#### DashboardPanel 续写入口
- `client/src/components/workspace/DashboardPanel.tsx`
  - `handleContinue` 改为调用 `POST /api/pipeline/start`（mode: "continue"）
  - 按钮文案从"续写下一章"改为"智能续写"
  - 描述文案更新为"从已有章节提取资产、规划卷纲章纲，每个阶段可确认调整后继续创作"
  - 英文清理："Pipeline" → "流水线"（2 处）

#### PipelinePage 支持 continue 模式
- `client/src/pages/PipelinePage.tsx`
  - `defaultPhases` 映射：`"continue"` 使用与 standalone 相同的阶段列表

### 三、中文化收尾

- `client/src/utils/translate.ts`
  - `translatePipelineStatus` 补充 `paused` 和 `error` 状态

## 调用链路

### 续写 Pipeline 流程
```
DashboardPanel "智能续写" 
  → POST /api/pipeline/start { mode: "continue" }
  → executePipeline → executeAnalyzePhase_continue
    → analyzeInput（分析已有章节）
    → decomposeIntoAssets（拆解入库）
    → generateOutlineFromChapters（推断大纲）
  → 暂停等待确认
  → confirmPhase → assetsPhase（人物/世界观/风格）
  → 暂停等待确认
  → confirmPhase → planningPhase_standalone（卷纲）
  → 暂停等待确认
  → confirmPhase → chapterOutlinesPhase（章纲）
  → 暂停等待确认
  → confirmPhase → consistencyPhase（一致性检查）
  → 暂停等待确认
  → confirmPhase → writingPhase（正文生成 + StoryState + 记忆提取）
```

## 风险说明
- 旧续写路由 `/api/novels/:id/continue` 保留但不再被前端调用
- assetsPhase/planningPhase/chapterOutlinesPhase/consistencyPhase 无需修改，已有"已有则跳过"逻辑
- 续写模式和自主创作共用相同的 Pipeline 阶段框架，只是第一个阶段的输入源不同
