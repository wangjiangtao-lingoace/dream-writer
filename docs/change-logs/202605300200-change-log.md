# 变更记录：Pipeline 一致性与质量提升

**日期**: 2026-05-30
**分支**: feat/lightweight-rebuild
**变更原因**: 用户反馈流水线生成的角色、世界观、风格等内容与主题主线不一致，创作过程中容易"写飘"——前后逻辑不一致、人物错乱。

## 修改点

### 1. generators.ts — 6 项生成器 prompt 和参数改动

1. **generateStyle()** 增加 `worldview` 和 `characters` 参数，prompt 中注入世界观和人物设定，约束风格与世界观时代背景、人物性格匹配
2. **generateWorldview()** maxTokens 从 1500 提升至 3000，避免世界观描述被截断
3. **generateEnrichedChapterOutlines()** 传入完整人物档案 JSON（身份、性格、能力、动机）和完整世界观规则，替代之前只传角色名的做法
4. **generateStoryArcs()** 修复 bug：`style` 参数在函数签名中但未写入 prompt，现已注入风格要求
5. **generateConsistencyCheck()** 增加 `outline`、`worldview`、`characters`、`style` 四个可选参数，一致性检查维度从 7 个扩展到 10 个（新增人物一致性、世界观一致性、风格一致性）
6. **所有 8 个生成器** 的 system prompt 末尾追加通用一致性约束指令

### 2. consistencyPhase.ts — 加载核心资产传入一致性检查

- `executeConsistencyCheckPhase()` 新增从 KnowledgeAsset 表加载 outline/worldview/character/style 四个核心资产
- 将核心资产传给 `generateConsistencyCheck()`，使一致性检查能验证人物、世界观、风格维度

### 3. pipelineUtils.ts — 移除硬编码 fallback

- `buildFallbackWorldview()` 移除古装悬疑硬编码（"王府、京城、朝堂、边境"、"验尸知识"），改为通用空模板
- `buildFallbackStyle()` 移除硬编码的"快节奏推进"等假设，改为通用空模板

### 4. PipelineService.ts + assetsPhase.ts + planningPhase.ts — 更新调用签名

- `regenerateStep()` 中 `generateStyle()` 调用增加 worldview 和 characters 加载
- `regenerateStep()` 中 `generateConsistencyCheck()` 调用增加核心资产加载
- `executeAssetsPhase()` 中 `generateStyle()` 调用增加世界观和人物参数
- `executePlanningPhase()` 中 `generateStyle()` 调用增加世界观和人物参数

## 风险说明

- 所有改动仅影响生成器的 prompt 内容和参数传递，不涉及数据库 schema 变更
- `generateConsistencyCheck()` 新增参数均为可选，向后兼容
- `generateStyle()` 新增两个必选参数，已更新所有调用点
- maxTokens 提升会略微增加 token 消耗
