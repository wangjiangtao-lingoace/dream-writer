# 202607030040 - 编辑 Agent 润色阶段

## 变更原因
在 Pipeline 写作阶段中新增编辑 Agent 润色步骤，对 AI 生成的章节正文进行二次润色，使其更像人类写作，提升文本质量。

## 修改点
1. **新增 `generateEditorPolish` 函数**：在 `generators.ts` 中新增编辑润色生成器，使用资深文学编辑角色的 system prompt，以 temperature=0.4 的低随机性进行润色，保持原文剧情和人物不变，优化句式多样性、去除 AI 味表达、增加感官描写。
2. **在写作流程中插入润色步骤**：在 `writingPhase.ts` 的 `generateInitialChapterDrafts` 函数中，章节正文生成完成后、质量检测之前，插入编辑润色调用。
3. **环境变量控制**：通过 `ENABLE_EDITOR_POLISH` 环境变量控制是否启用（默认启用，设为 `false` 可关闭）。

## 文件列表
| 文件 | 变更说明 |
|------|---------|
| `server/src/services/pipeline/generators.ts` | 新增 `generateEditorPolish` 导出函数，接收正文内容和上下文（章纲、人物、风格），返回润色后的文本 |
| `server/src/services/pipeline/writingPhase.ts` | 导入 `generateEditorPolish`；在 `generateInitialChapterDrafts` 中 draft 生成后、quality validation 前插入润色调用，含 try-catch 容错和长度校验 |

## 风险说明
- **额外 LLM 调用**：每章增加 1 次 LLM 调用（润色），会增加 token 消耗和延迟。可通过 `ENABLE_EDITOR_POLISH=false` 关闭。
- **润色质量不可控**：LLM 润色可能改变原文语义或引入错误。已通过长度校验（润色结果必须 > 原文 50%）和 try-catch 容错降低风险。
- **不影响重试循环**：润色在质量检测之前执行，若质量不合格触发重试，重试后的 draft 不会再次润色（符合预期：重试 draft 已针对质量问题修正）。
