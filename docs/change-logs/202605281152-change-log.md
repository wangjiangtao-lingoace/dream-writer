# 变更记录

## 变更原因
新增「导入续写」功能：支持上传 txt 小说文本，自动提取人物、世界观、剧情、写作风格等结构化数据，落库后可直接续写后续章节。同时支持拆书和自主创作两种模式的续写。

## 修改点

### 新建文件
- `server/src/services/ImportAnalysisService.ts` — txt 文本分析与结构化提取服务，支持章节拆分（正则+字数回退）、4 路并发 LLM 提取（人物/世界观/剧情/风格）、完整数据落库
- `server/src/services/ContinuationService.ts` — 续写服务，加载完整上下文（人物/世界观/钩子/伏笔/主线/记忆/风格/剧情状态/最近章节），生成大纲+正文，自动更新 StoryState 和提取记忆
- `server/src/routes/continuation.ts` — API 路由：`POST /api/import/analyze`（导入分析）和 `POST /api/novels/:id/continue`（续写章节）
- `client/src/pages/ImportContinue.tsx` — 导入续写前端页面：txt 上传/粘贴 → 分析中 → 完成跳转工作台

### 修改文件
- `server/src/app.ts` — 注册 continuation 路由
- `client/src/router/index.tsx` — 新增 `/create/import` 路由
- `client/src/pages/CreateWork.tsx` — 新增「导入续写」入口卡片（2列→3列布局）
- `client/src/components/workspace/DashboardPanel.tsx` — 新增「续写下一章」按钮（已有草稿章节时显示）
- `client/src/components/workspace/WritePanel.tsx` — 章节列表头部新增「续」按钮，一键续写并自动选中新章节

## 风险说明
- 不影响现有接口和业务逻辑，纯新增功能
- ImportAnalysisService 的 LLM 提取使用 temperature=0.3，确保提取准确性
- ContinuationService 使用 10 条铁律系统 prompt，与 pipeline 写作质量一致
- 章节拆分支持标准格式（第X章）和无标记文本（按字数切分），兼容性强

## 验证情况
- TypeScript 全量编译通过（shared + server + client）
