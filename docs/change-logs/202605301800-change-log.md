# 百万字长篇可持续性修复（第二轮）：一致性存储 + 记忆管理 + 钩子伏笔上下文 + 层级摘要 + 宏观结构 + 断点续写

## 变更原因

第一轮修复（RAG+状态追踪+章纲修正+字数）后，仍有 6 个维度未覆盖：
1. 一致性检查结果只存 PhaseResult JSON，ConsistencyIssue 表从未写入
2. Pipeline 写作模式未调用 autoManageMemories，记忆无限增长
3. ContextAssembler 未加载活跃钩子和未回收伏笔，写作时看不到悬念状态
4. buildPreviousVolumeSummary 逐章拼接所有卷，上下文线性增长
5. plotStructure 只有 4 阶段，无法支撑百万字长篇的宏观节奏
6. 写作循环单章失败即中断整批，无断点续写能力

## 改动

### 1. ConsistencyIssue 表存储

`server/src/services/pipeline/consistencyPhase.ts`：
- 一致性检查完成后，将 `result.issues` 数组写入 ConsistencyIssue 表
- 每条 issue 包含 type、severity、description、evidence（相关章节）、suggestion

### 2. Pipeline 写作调用 autoManageMemories

`server/src/services/pipeline/writingPhase.ts`：
- 导入 `autoManageMemories`，每章后处理时调用
- 管理记忆生命周期、整合相似记忆、清理低重要性记忆

### 3. ContextAssembler 加载 Hook/Foreshadow

`server/src/services/pipeline/contextAssembler.ts`：
- 新增 `loadActiveHooks()`：加载 status=planted/active 的钩子，最多 8 条
- 新增 `loadPlantedForeshadows()`：加载 status=planted 的伏笔，最多 8 条
- `assembleForChapter` 并行加载扩展到 10 项
- `buildCompactContext` 新增【未解决钩子】和【未回收伏笔】两个区块

### 4. 卷纲承接层级摘要压缩

`server/src/services/pipeline/chapterOutlinesPhase.ts`：
- `buildPreviousVolumeSummary` 改为层级摘要：
  - 距离 >= 2 的远卷：只传卷级摘要（~50 tokens）
  - 相邻卷：传首尾章节 + 关键转折（~150 tokens）
- 5 卷小说的前序摘要从 ~600 tokens 压缩到 ~200 tokens

### 5. 宏观结构规划增强

`server/src/services/pipeline/generators.ts`：
- plotStructure 从 4 阶段扩展为 8 阶段：
  - setup（开篇建立 0-5%）→ rising_action（上升 5-20%）→ first_climax（第一高潮 20-35%）
  → deepening（深度发展 35-55%）→ major_turning（重大转折 55-70%）
  → escalation（冲突升级 70-85%）→ final_climax（最终高潮 85-95%）→ resolution（收束 95-100%）
- 每阶段要求至少 3 个具体事件
- maxTokens 从 3000 → 4000

### 6. 写作断点续写机制

`server/src/services/pipeline/writingPhase.ts`：
- 写作循环中每章独立 try/catch，单章失败不中断整批
- 失败章节记录到 failedChapters 数组，批次完成后汇总报告
- 每章完成后更新 PipelineJob.currentStep 作为 checkpoint
- 部分章节成功时仍标记 job 为 completed（而非 error）
- 已有章节跳过机制天然支持 resume（重跑时自动跳过已有内容的章节）

| `server/src/services/pipeline/contextAssembler.ts` | 加载活跃钩子/伏笔到写作上下文 |

## 验证

- TypeScript 类型检查：server + client 均通过
- ConsistencyIssue：无 issue 时不写入，不影响流程
- autoManageMemories：异常被 catch，不影响主流程
- Hook/Foreshadow 无数据时返回空数组，不影响上下文
- 层级摘要：第 1 卷返回空字符串，与原逻辑一致
- 断点续写：已有章节自动跳过，失败章节不阻塞后续

## 风险说明

- **autoManageMemories 的 LLM 调用**：每章多一次记忆整合调用，增加约 200 tokens 成本
- **plotStructure 8 阶段**：LLM 输出更长，可能增加大纲生成时间约 2-3 秒
- **ConsistencyIssue 批量写入**：使用 createMany，大量 issue 时性能优于逐条插入
- **断点续写的 checkpoint 粒度**：currentStep 级别，进程崩溃后 resume 从整批重跑（已有章节自动跳过）
