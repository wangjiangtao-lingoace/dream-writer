# 变更记录 2026-05-27 11:50

## 变更原因

RAG（检索增强生成）Phase 2 基础设施搭建。需要 sqlite-vec 向量存储层作为知识检索的底层能力。

## 修改点

1. 安装 `sqlite-vec` npm 包（v0.1.9），提供 SQLite 向量搜索扩展
2. 新增 `server/src/db/vectorStore.ts` 模块，封装 sqlite-vec 虚拟表 CRUD 操作

| `server/src/db/vectorStore.ts` | 新增 | 向量存储层，管理 vec0 虚拟表、向量插入、KNN 搜索、按 owner/novel 删除、统计 |

## 设计说明

- 使用独立的 better-sqlite3 连接（非 Prisma），因为 Prisma 不暴露 `loadExtension()` API
- 两个连接指向同一个 `dev.db` 文件
- 复用 `prisma.ts` 的数据库路径解析逻辑
- 优雅降级：sqlite-vec 加载失败时 `available = false`，所有操作变为 no-op
- 单例模式：`getVectorStore()` 保证全局唯一实例

## 风险说明

- sqlite-vec 需要平台特定的预编译二进制（darwin-arm64/x64、linux-x64/arm64、win32-x64），不支持的平台会降级
- better-sqlite3 连接与 Prisma 连接同时打开同一个 SQLite 文件，依赖 SQLite 的文件级锁保证一致性
- vec0 虚拟表不支持 Prisma migration，通过 `CREATE VIRTUAL TABLE IF NOT EXISTS` 自动创建
