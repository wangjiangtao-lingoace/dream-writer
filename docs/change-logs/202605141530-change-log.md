# 变更记录：一键安装/停止脚本

## 变更原因
新用户上手门槛高，需要手动安装 Node.js、pnpm、初始化数据库、构建 shared 包等多个步骤。需要一键脚本让新用户快速搭建环境并启动项目，同时提供停止脚本方便关闭服务。

## 修改点
- 新增 `scripts/setup.sh`：macOS 一键安装脚本，自动检测/安装依赖、clone 代码、初始化项目
- 新增 `scripts/stop.sh`：停止前端（5173）和后端（3000）服务

## 文件列表
| 文件 | 说明 |
|------|------|
| `scripts/setup.sh` | 新增：macOS 一键安装脚本 |
| `scripts/stop.sh` | 新增：停止服务脚本 |

## 使用方式
```bash
# 安装并启动
curl -fsSL https://raw.githubusercontent.com/wangjiangtao-lingoace/dream-writer/feat/lightweight-rebuild/scripts/setup.sh | bash

# 停止服务
cd ~/dream-writer && bash scripts/stop.sh
```

## setup.sh 流程
1. 检测/安装 Xcode CLT（提供 git）
2. 检测/安装 Node.js >= 20.19.0（通过 nvm）
3. 检测/安装 pnpm >= 10.6.0
4. clone 代码到 `~/dream-writer`（已存在则 pull 更新）
5. 从 `.env.example` 生成 `.env`
6. `pnpm install` → `prisma:generate` → `prisma:push` → `build:shared`
7. 询问是否立即启动 `pnpm dev`

## stop.sh 流程
1. 查找并终止端口 5173（前端 Vite）上的进程
2. 查找并终止端口 3000（后端 Express）上的进程

## 前提条件
- GitHub 仓库需设为公开（用户通过 HTTPS clone）
- 仅支持 macOS

## 风险说明
- nvm 安装会修改 shell profile（~/.zshrc 或 ~/.bashrc）
- stop.sh 使用 `kill -9` 强制终止，未保存的数据可能丢失
- 脚本使用 `set -euo pipefail`，任何步骤失败会立即停止
