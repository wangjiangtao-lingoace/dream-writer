# 变更记录

## 变更原因
UI/UX 现代化第三轮：统一错误/成功通知方式，修复 ConfirmDialog 暗色主题适配。将所有 `alert()` 替换为 sonner toast 通知，提升用户体验。

## 修改点
1. **CommonComponents.tsx**: ConfirmDialog 从硬编码亮色样式改为 CSS 变量暗色主题，使用语义色变量替代 `#dc2626`/`#8b4513` 等硬编码值
2. **toast.tsx**: Toaster 样式更新为正确的 CSS 变量名
3. **MainlinePanel.tsx**: 3 处 `alert()` → `toast.error()`/`toast.success()`
4. **WritePanel.tsx**: 1 处 `alert()` → `toast.success()`
5. **HookPanel.tsx**: 3 处 `alert()` → `toast.error()`/`toast.success()`
6. **NovelWorkspace.tsx**: 4 处 `alert()` → `toast.error()`
7. **AnalyzeCreate.tsx**: 6 处 `alert()` → `toast.error()`

## 风险说明
- toast 通知比 alert() 更轻量，不会阻塞用户操作
- 验证类错误（如"请输入标题"）使用 toast.error()，成功类使用 toast.success()
- 仍有 5 处 `confirm()` 调用未替换（需添加 ConfirmDialog 状态管理，后续优化）
