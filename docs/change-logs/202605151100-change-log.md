# 变更记录 2026-05-15 11:00

## 变更原因

实现阶段一 Task 2：通用搜索API集成，用正规搜索API替代脆弱的直接抓取方式。

## 修改点

### 新建文件
- `server/src/services/SearchProviderService.ts` — 搜索Provider抽象层
- `server/src/services/ContentExtractionService.ts` — 内容提取服务

### 修改文件
- `server/src/routes/search.ts` — 集成新服务，重构搜索流程
- `server/.env.example` — 添加搜索配置

### 功能变更

1. **SearchProviderService（搜索Provider抽象层）**
   - 定义 `SearchResult` 和 `SearchProvider` 接口
   - 实现 `BingSearchProvider` — 调用 Bing Web Search API
   - 实现 `GoogleSearchProvider` — 调用 Google Custom Search API
   - 实现 `SogouScrapingProvider` — 包装搜狗抓取逻辑
   - 按优先级顺序尝试Provider（Bing → Google → Sogou）
   - 优雅处理Provider错误（配额超限、网络失败 → 降级到下一个）

2. **ContentExtractionService（内容提取服务）**
   - 使用可读性启发式算法提取网页主要内容
   - 按文本块评分：长度、中文字符密度、小说相关关键词
   - 移除样板内容（页头、页脚、导航、广告）
   - 返回结构化 `{ title, author, synopsis, bodyText, confidence }`

3. **search.ts 重构**
   - 替换原有硬编码5个站点的抓取方式
   - 新流程：`SearchProviderService.search` → `ContentExtractionService.extract` → 验证质量 → 返回最佳结果
   - 保留已知来源URL作为降级方案
   - 保持API响应格式兼容
   - 清理约200行死代码（`generateSynopsis`, `guessGenre` 等）

4. **环境配置**
   - `SEARCH_PROVIDER_PRIORITY=bing,google,sogou`
   - `BING_SEARCH_API_KEY`
   - `GOOGLE_SEARCH_API_KEY`
   - `GOOGLE_SEARCH_CX`

## 风险说明

- 搜索结果URL未做SSRF验证（低风险，URL来自搜索引擎）

## 使用说明

配置环境变量后即可使用：
```bash
# 在 server/.env 中添加
SEARCH_PROVIDER_PRIORITY=bing,google,sogou
BING_SEARCH_API_KEY=your_key_here
GOOGLE_SEARCH_API_KEY=your_key_here
GOOGLE_SEARCH_CX=your_cx_here
```

API会按优先级自动尝试各Provider，失败时自动降级到下一个。
