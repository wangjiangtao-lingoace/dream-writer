## 1. 变更原因

`dream-writer` 仓库虽然定位为「AI-Novel 的轻量重写版」，但实际是 **AI-Novel 旧仓库的完整克隆**：

- `server/prisma/schema.prisma` 1997 行 / 70+ 表
- `server/src/` 含 `agents/`、`services/`（小说/世界/角色/章节导演/任务/风格引擎/状态/RAG 等 9 个子领域）、`prompting/`（70+ prompt 资产）、`routes/`（45 条路由）
- `client/src/` 含 9 个页面 + 8 个 shadcn UI 组件 + `ancient/` 双子目录
- 既有 `IMPLEMENTATION_ROADMAP.md` 仍是「12 周完整移植」的旧节奏

这与「先打通骨架再分阶段堆能力」的轻量诉求冲突。决定采取 **外科手术式减法**（Plan B）：保留 LLM 调用层、UI 主题、路由壳、shared 类型四块基础设施；删除所有业务实现；重写 schema、`app.ts`、首页与路由表，跑通 `/api/health`。
