# 变更记录 202606201500

## 变更原因
系统全方位审核发现三大缺陷：Token 消耗完全无追踪、AI 辅助写作按钮未实现、编辑器功能基础。本次变更一次性补齐。

## 修改点

### 1. Token 消耗追踪（P0）
- Prisma schema `GenerationLog` 表新增 6 个字段：`promptTokens`、`completionTokens`、`totalTokens`、`cost`、`provider`、`model`
- `LlmInvokeService.doCompleteText` 返回类型从 `string` 改为 `CompletionResult`，提取 API 响应中的 `usage` 字段
- 新增 `completeTextWithUsage` 方法（返回完整结果含 token 用量）
- 新增 `persistGenerationLog` 方法（fire-and-forget 写入日志，不阻塞主流程）
- `completeText` 保持向后兼容（返回 `string | null`）
- `CharacterImportService` 适配 `completeTextOrThrow` 新返回类型

### 2. AI 辅助写作按钮（P1）
- 新增 `POST /api/ai/polish` 接口：支持 `deai`（去AI味）和 `enhance`（增强压迫）两种模式
- 新增 `POST /api/ai/continue-chapter` 接口：从当前章节末尾续写
- `NovelWorkspace` 的 `handleToolbarAction` 实现实际 API 调用（替换原 toast.info 占位）
- `RichTextEditor` 工具栏新增"续写"按钮，AI 按钮在处理中自动禁用
- `ToolbarButton` 组件新增 `disabled`、`title` 属性支持
- `WorkspaceWriteLayout` 透传 `aiProcessing` 状态

### 3. 编辑器增强（P2）
- 新增撤销/重做按钮（工具栏最左侧）
- 新增查找替换面板（Ctrl+F 触发，支持区分大小写、替换、全部替换）
- 新增全屏沉浸模式（Ctrl+Shift+F 切换，ESC 退出，半透明遮罩 + 800px 居中编辑器）
- 工具栏按钮新增 `title` 提示
- 添加工具栏分隔线区分功能区域

## 风险说明

- **向后兼容**：`completeText` 返回类型不变，所有 50+ 调用点零改动；仅 `CharacterImportService` 一处调用 `completeTextOrThrow` 需适配
- **非阻塞日志**：`persistGenerationLog` 使用 `.catch(() => {})` 确保日志写入失败不影响主流程
- **cost 字段留空**：后续任务实现费用计算
- **Prisma db push 失败**：已通过直接 SQL ALTER TABLE 绕过，schema 与数据库一致
