# 变更记录

## 变更原因
移除代码中所有硬编码 API Key 兜底逻辑，确保未配置密钥时功能不可用，防止密钥泄露。

## 修改点

### 1. crypto.ts — 移除硬编码加密密钥
- `server/src/utils/crypto.ts`
  - 删除兜底密钥 `"dream-writer-dev-only-encryption-key-32b"`
  - `AI_CONFIG_ENCRYPTION_KEY` 未配置时直接抛错，不再静默降级

### 2. LlmInvokeService.ts — 移除所有兜底生成
- `server/src/services/llm/LlmInvokeService.ts`
  - 删除 `buildFallbackDraft()` 函数（生成假章节内容）
  - 删除 `streamFallback()` 方法（流式假章节）
  - 删除 `streamFallbackRaw()` 方法（流式假错误信息）
  - `streamText()` 未配置 API Key 时抛 `LlmError`，不再 yield 兜底文本
  - `streamChapterDraft()` 未配置 API Key 时抛 `LlmError`，不再 yield 兜底草稿
  - 流式请求失败时抛 `LlmError`，不再降级到兜底

### 3. AIService.ts — 移除兜底返回值
- `server/src/services/AIService.ts`
  - `generateInspiration()`: `result || "生成失败"` → 抛错
  - `generateVolumeOutline()`: 同上
  - `generateChapterOutline()`: 同上
  - `generateChapterOutlineForChapter()`: 同上
  - `generateChapterContent()`: 同上
  - `checkConsistency()`: `result || "校验失败"` → 抛错

### 4. DirectorService.ts — 移除兜底返回值
- `server/src/services/DirectorService.ts`
  - `analyze()`: `result || "分析失败"` → 抛错
  - `planPlot()`: `result || "规划失败"` → 抛错
  - `simulateReader()`: `result || "模拟失败"` → 抛错

### 5. BookAnalysisService.ts — 移除兜底拆书
- `server/src/services/BookAnalysisService.ts`
  - 删除 `fallbackSection()` 函数（本地假拆书分析）
  - 分区分析 LLM 返回 null 时抛错，不再使用本地兜底
  - 分区分析失败时存储错误提示，不再存储假分析内容

## 验证情况
- TypeScript 全量编译通过（server）
- 全局搜索确认无残留兜底模式

## 风险说明
- **必须配置**：`AI_CONFIG_ENCRYPTION_KEY`（server/.env，至少 32 字符）+ 至少一个 LLM provider 的 API Key
- 未配置时所有 AI 功能（生成灵感、大纲、正文、拆书、风格分析等）会抛出明确错误
- `.env` 文件不在 git 中，不会泄露
