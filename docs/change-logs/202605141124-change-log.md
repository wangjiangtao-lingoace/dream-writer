# 变更记录：修复世界观跨作品共享 Bug

## 变更原因
用户报告不同小说的世界观数据被共享。经排查，worldviews 路由存在 3 个 novelId 处理缺陷，导致世界观数据跨作品泄露。

## 修改点
1. `GET /api/worldviews` 路由忽略 novelId 查询参数，返回所有作品的世界观 → 增加 where 过滤
2. `POST /api/worldviews` 路由的 Zod schema 缺少 novelId 字段，Zod 静默剥离未知字段导致 novelId 为 null → schema 增加 novelId 必填字段
3. `AnalysisToNovelService.saveToNovel()` 创建世界观时未传入 novelId 参数 → 补充 novelId

## 文件列表
| 文件 | 说明 |
|------|------|
| `server/src/routes/worldviews.ts` | GET 增加 novelId 查询过滤；POST schema 增加 novelId 字段 |
| `server/src/services/AnalysisToNovelService.ts` | worldview.create 补充 novelId 参数 |

## 风险说明
- 数据库中可能存在 novelId=null 的孤立世界观记录（历史数据），需手动清理或重新关联到对应作品
- Characters 路由已正确使用 novelId 过滤，无此问题
