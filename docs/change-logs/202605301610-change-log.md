# 创作质量约束：语言规范 + 大纲深度 + 章纲独立性

## 变更原因

Pipeline 生成的内容存在以下质量问题：
1. **AI 味重**：生成文本中频繁出现"不禁"、"宛如"、"缓缓"等 AI 标志性词汇
2. **大纲深度不足**：plotStructure 只有一句话概括，无法支撑百万字长篇创作
3. **卷纲描述空洞**：keyEvents 少、转折点抽象、钩子不具体，无法实际指导写作
4. **章纲不可执行**：goal 写"推进剧情"、conflict 写"面临挑战"、hook 写"留下悬念"，写手无法据此独立创作
5. **水字数**：无意义的环境渲染、重复表达、为凑字数扩展描写

## 改动

### 1. generators.ts — 4 个生成函数注入质量约束

**generateOutline**（大纲生成）：
- 新增语言要求：通俗白话，禁止 AI 味词汇和空洞修饰语
- 新增质量要求：大纲必须支撑 100 万字以上长篇，plotStructure 每阶段至少 3 个具体事件，protagonist/antagonist 需完整动机链

**generateVolumeOutline**（卷纲生成）：
- 新增语言要求：禁止 AI 味词汇和"本卷将探索…"类套话
- 新增卷纲质量要求：keyEvents 至少 5 个具体事件，turningPoint/climax/endHook 必须有具体内容，卷间必须有承接关系

**generateChapterOutlines**（章纲生成，基础版）：
- 新增语言要求：通俗白话，禁止 AI 味词汇
- 新增章纲独立性要求：goal/conflict/hook 三个字段必须包含具体信息，脱离上下文也能据此写出完整章节

**generateEnrichedChapterOutlines**（章纲生成，详细版）：
- 同上，增加 characters 字段要求

### 2. writingPhase.ts — 写作铁律增强

在已有的 10 条写作铁律基础上新增 5 条质量铁律：
- 语言通俗白话，不要书面腔和翻译腔
- 禁止水字数（无意义环境渲染、重复表达、凑字数扩展）
- 每段必须有信息增量（推进情节/揭示人物/制造氛围）
- 描写必须符合当前场景和作品风格
- 禁止万能句式（"他感到一阵xxx"、"一股xxx的感觉涌上心头"）

### 3. analyzePhase.ts — 拆解和续写大纲

**decomposeOutline**：新增语言保留要求，原文口语化表达保持原样

**generateOutlineFromChapters**：新增语言要求 + 质量要求（支撑百万字长篇）

## 修改文件

| 文件 | 改动 |
|------|------|
| `server/src/services/pipeline/generators.ts` | 4 个生成函数的 system prompt 注入质量约束 |
| `server/src/services/pipeline/writingPhase.ts` | 写作铁律从 10 条扩展到 15 条 |
| `server/src/services/pipeline/analyzePhase.ts` | 2 个大纲函数的 system prompt 注入语言和质量要求 |

## 验证

- TypeScript 类型检查：server + client 均通过
- prompt 改动不影响类型系统，仅影响 LLM 输出质量

## 风险说明

- **生成 token 增加**：更详细的约束可能导致 LLM 输出更长，但 maxTokens 已有上限控制
- **风格一致性**：新增的语言约束与已有的写作铁律方向一致，不会冲突
