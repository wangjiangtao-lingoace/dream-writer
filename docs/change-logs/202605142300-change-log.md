# 变更记录：独立创作模式支持

## 变更原因
用户选择"独立创作"模式时，创作总控台显示大量"缺失项"（缺拆书、缺仿写方案等），且"自动生成"按钮被禁用。原因是总控台的流程检查和按钮逻辑只适配了"拆书仿写"模式，没有为独立创作场景提供直达 Pipeline 的路径。

## 修改点
1. 后端 `workflow-status` 接口增加 `creationMode` 字段，区分"独立创作"和"仿写"两种模式
2. 独立创作模式下，缺失项检查跳过"拆书/仿写方案"，仅提示 Pipeline 可自动生成的资产（人物卡、世界观、风格配置等），并标注"将自动生成"
3. `nextActions` 新增"从灵感一键生成全部"动作，当有灵感但无拆书/仿写方案时可直接启动 Pipeline
4. 前端总控台适配独立创作模式：卡片展示灵感/资产/章节/流程状态，任务关系图改为"灵感 → AI 规划 → 章节创作"三节点

## 文件列表
| 文件 | 说明 |
|------|------|
| `server/src/routes/novels.ts` | workflow-status 增加 creationMode 判断、standalone 缺失项逻辑和 nextAction |
| `client/src/pages/NovelWorkspace.tsx` | WorkflowStatus 类型增加 creationMode；handleAction 支持 standalone；卡片和关系图适配独立创作模式 |

## 风险说明
- 独立创作模式下 Pipeline 从 `novel.inspiration` 生成全部资产，灵感为空时 standalone 按钮不可用，需用户先填写灵感
- 已有拆书/仿写方案的用户自动归为 imitation 模式，不受影响
