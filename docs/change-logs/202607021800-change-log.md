# 变更记录

**变更时间**: 2026-07-02 18:00

## 变更原因
StoryState 是剧情系统的中央控制台，但状态更新依赖手动调用或 LLM 后处理，缺少事件驱动的自动状态流转机制。需要增加状态机规则引擎，让剧情阶段在满足条件时自动推进。

## 修改点
1. 在 `StoryStateService.ts` 中新增 `autoTransitionState()` 函数，实现状态流转规则引擎
2. 定义三条自动流转规则：setup→development、development→climax、climax→resolution
3. 在 `postProcessing.ts` 的 `mergedPostProcessing()` 尾部调用状态流转检查

## 文件列表
| 文件 | 变更说明 |
|------|----------|
| `server/src/services/StoryStateService.ts` | 新增 `TransitionRule`/`TransitionContext`/`TransitionResult` 类型，`TRANSITION_RULES` 规则数组，`autoTransitionState()` 函数 |
| `server/src/services/pipeline/postProcessing.ts` | 导入 `autoTransitionState`，在 `mergedPostProcessing()` 末尾调用（非阻塞） |

## 流转规则
| 流转方向 | 触发条件 | 说明 |
|----------|----------|------|
| setup → development | currentChapter > 3 | 前3章完成后自动进入发展阶段 |
| development → climax | tensionAccumulation > 7 且 pleasureCooldown <= 0 | 压抑值累积超过阈值且爽点冷却结束 |
| climax → resolution | 所有伏笔回收且 mainConflict 非空 | 伏笔全部回收且主线有推进 |

## 风险说明
- 自动流转是"建议"机制，`mergedPostProcessing` 中 LLM 返回的 `currentPhase` 会先写入数据库，自动流转在之后检查，不会覆盖 LLM 判断
- 如需跳过自动流转，可在后处理前手动设置 `currentPhase`，规则只会在匹配 `from` 阶段时触发
- 规则硬编码在 `TRANSITION_RULES` 数组中，如需扩展直接添加新规则即可
