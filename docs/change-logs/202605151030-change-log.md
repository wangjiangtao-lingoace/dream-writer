# 变更记录 2026-05-15 10:30

## 变更原因

实现阶段一 Task 1：AnalyzeCreate 页面文本粘贴功能，解决拆书时查不到数据源的问题。

## 修改点

### 修改文件
- `client/src/pages/AnalyzeCreate.tsx`: 添加文本粘贴功能

### 功能变更

1. **新增状态变量**
   - `pastedText`: 用户粘贴内容
   - `sourceMode`: 数据来源追踪（"none" | "search" | "manual"）

2. **新增 resolveSourceMaterial() 函数**
   - 优先使用粘贴文本（>=80字符）
   - 否则使用搜索结果

3. **输入步骤UI增强**
   - 标题输入框下方添加 textarea
   - 显示字数统计（"已输入 X 字 / 最多 12,000 字"）
   - 粘贴 >=80 字时显示"直接拆书"按钮

4. **预览步骤增强**
   - 搜索失败时内联显示 textarea
   - 用户可直接粘贴文本，无需返回输入步骤
   - 粘贴 >=80 字后激活"开始拆解分析"按钮

5. **handleAnalyze() 重构**
   - 使用 resolveSourceMaterial() 解析数据源
   - 优先使用粘贴文本，否则使用搜索结果

## 验证情况

- [ ] 在 AnalyzeCreate 页面粘贴 >= 80 字文本，确认可直接进入拆书分析
- [ ] 粘贴 < 80 字文本，确认仍需搜索
- [ ] 搜索失败后，确认可在预览步骤粘贴文本

## 数据流

```
用户粘贴文本(>=80字) → resolveSourceMaterial() → POST /api/book-analysis { sourceText } → BookAnalysisService.clipSourceText()(12000字限制) → 8维分析
```
