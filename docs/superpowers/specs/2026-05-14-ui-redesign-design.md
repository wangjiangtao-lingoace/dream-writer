# Dream Writer UI 全面重构设计文档

## 目标

将 Dream Writer 从古风主题改造为深色科技风，实现 AI 主导 + 人工审核的智能化工作流，使应用可开源部署（用户自带 API Key）。

## 设计决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 布局 | 三栏：侧边栏 + 主内容 + AI 面板 | AI 面板常驻，上下文感知，效率最高 |
| 配色 | 炭黑琥珀（纯黑底 + 橙色主色） | 区别于千篇一律的蓝紫，创作氛围温暖 |
| 侧边栏 | 固定文字侧边栏（200px，可折叠） | 分组清晰，学习成本低 |
| 工作台 | 三栏：目录 \| 编辑器 \| AI 面板 | 编辑器和 AI 并排，减少切换 |
| AI 面板 | 上下文操作面板 | 根据当前状态动态推荐操作，带快捷键 |
| 书架 | 列表视图 | 信息密度高，适合多部作品管理 |
| Pipeline | 阶段流 + 章节列表 | 横向进度一目了然，章节状态实时更新 |
| API Key | 首次弹窗 + 首页横幅 + 设置页 | 三重引导，确保用户能配置 |

### 3.1 书架首页

├─────────────────────────────────────────────────┤
│ ┌─────────────────────────────────────────────┐ │
│ │ [封面] 标题 · 摘要 · 状态 · 进度条 · 字数    │ │
│ ├─────────────────────────────────────────────┤ │
│ │ [封面] 标题 · 摘要 · 状态 · 进度条 · 字数    │ │
│ └─────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────┘
```

- **配置横幅**：未配置 API Key 时显示，点击弹出配置弹窗
- **列表项**：封面缩略图（32x42）+ 标题 + 摘要（1行截断）+ 状态标签 + 进度条 + 字数
- **状态**：进行中（绿色）、草稿（灰色）、已完成（橙色）
- **新建按钮**：右上角橙色按钮

### 3.2 创建作品

**布局**：两步向导

**Step 1：基本信息**
- 标题、类型、灵感描述、封面上传

**Step 2：AI 生成（核心差异化）**
- 输入标题+类型后，AI 自动生成：
  - 大纲（可编辑）
  - 主要人物（可编辑/删除/添加）
  - 世界观设定（可编辑）
- 用户只需审核确认，不是从空白开始
- "重新生成"按钮允许 AI 重来

### 3.3 小说工作台

**布局**：三栏（目录 | 编辑器 | AI 面板）

```
┌──────────┬───────────────────────┬───────────────────┐
│ 章节目录  │  [正文] [大纲] [人物]  │  AI 助手           │
│          │                       │                   │
│ ▶ 第1章  │  ┌─────────────────┐  │  上下文：第3章正文  │
│   第2章  │  │                 │  │                   │
│   第3章  │  │   富文本编辑器    │  │  ┌─────────────┐  │
│   第4章  │  │                 │  │  │ ▶ 续写本章   │  │
│          │  │                 │  │  │   ⌘↵        │  │
│          │  │                 │  │  ├─────────────┤  │
│          │  │                 │  │  │ 🔍 一致性检查 │  │
│          │  │                 │  │  │   ⌘⇧C       │  │
│          │  │                 │  │  ├─────────────┤  │

│          │  │                 │  │  │   ⌘⇧P       │  │

- **左栏**：章节目录树，点击切换章节，拖拽排序
- **中栏**：tab 切换正文/大纲/人物视图，富文本编辑器
- **右栏**：AI 上下文操作面板

### 3.4 AI 面板交互

**核心机制**：根据当前上下文动态显示可用操作

| 上下文 | 可用操作 | 快捷键 |
|--------|----------|--------|
| 正文编辑 | 续写本章、一致性检查、润色优化、节奏分析 | ⌘↵ / ⌘⇧C / ⌘⇧P |
| 大纲视图 | 生成大纲、扩展大纲、优化结构 | ⌘⇧G |
| 人物视图 | 生成人物卡、人物关系分析、性格一致性 | ⌘⇧R |
| 空白章节 | AI 开篇、参考风格生成 | ⌘⇧O |

**操作流程**：
1. 用户点击操作（或按快捷键）
2. AI 面板显示流式输出（实时 token 渲染）
3. 输出完成后显示：采纳 / 修改 / 重新生成 按钮
4. 采纳 → 内容插入编辑器
5. 修改 → 弹出输入框，用户输入修改意见，AI 重新生成

**布局**：阶段流 + 章节列表

- **章节列表**：状态图标 + 章节标题 + 状态文字
- **人工审核节点**：AI 生成完成后暂停，等用户确认再继续下一阶段

### 3.6 AI 模型配置

**入口**：
1. 首次启动弹窗（模态）
2. 书架首页横幅（未配置时）
3. 侧边栏「设置 → AI 模型配置」

**配置项**：
- 提供商选择（下拉）：DeepSeek / OpenAI / Anthropic / Qwen / GLM / Kimi / Gemini / Mimo
- 模型选择（下拉）：根据提供商动态加载
- API Key（密码输入框，带显示/隐藏切换）
- Base URL（可选，用于自定义端点）
- 测试连接按钮

**存储**：localStorage（客户端），不传到后端（安全）

## 四、AI 工作流设计

### 4.1 核心原则

**AI 主导 + 人工审核**：
- AI 自动完成大部分工作（大纲、正文、一致性检查）
- 人类在关键节点审核确认
- 不是"点按钮生成"，而是"AI 在帮我写，我来把关"

### 4.2 智能化流程

**创建阶段**：
1. 用户输入标题+类型
2. AI 自动生成大纲/人物/世界观
3. 用户审核修改 → 确认

**写作阶段**：
1. AI 根据上下文推荐操作（续写/检查/润色）
2. 用户一键触发
3. AI 流式输出结果
4. 用户采纳/修改/重新生成

**批量生成阶段**：
1. 用户配置参数（章节数、字数）
2. AI 自动生成，阶段流实时显示
3. 每阶段完成后暂停，等用户确认
4. 用户可随时干预（暂停/取消/修改参数）

### 4.3 上下文感知

AI 面板根据以下因素动态调整推荐：
- 当前页面（书架/工作台/Pipeline）
- 当前视图（正文/大纲/人物）
- 编辑状态（空白/有内容/正在编辑）
- 历史操作（刚续写完 → 推荐一致性检查）
- 小说进度（初期 → 推荐大纲扩展；后期 → 推荐润色优化）

## 五、技术方案

### 5.1 CSS 架构

- **Design Token**：CSS 变量驱动，全局统一
- **组件样式**：外提为独立 CSS 文件，去掉内联 style
- **响应式**：CSS Grid + 媒体查询，支持 1280px+ 屏幕

### 5.2 布局组件

- `AppLayout`：三栏布局容器（侧边栏 + 主内容 + AI 面板）
- `Sidebar`：固定文字侧边栏，支持折叠
- `AIPanel`：右侧 AI 面板，上下文感知
- `TopBar`：顶部栏，Logo + 面包屑 + 用户操作

### 5.3 状态管理

- **服务端状态**：TanStack Query（已有）
- **AI 面板状态**：React Context（当前操作、输出内容、加载状态）
- **AI 配置**：后端数据库存储（见 5.5）
- **UI 状态**：React useState（侧边栏折叠、面板展开）

### 5.4 AI 流式输出

- 前端：EventSource / fetch + ReadableStream
- 后端：SSE（Server-Sent Events）
- 渲染：逐字渲染 + 光标动画

### 5.5 后端：AI 配置管理（BYOK）

**数据库表**：

```prisma
model AIConfig {
  id        String   @id @default(cuid())
  provider  String   // deepseek, openai, anthropic, qwen, glm, kimi, gemini, mimo
  model     String   // deepseek-chat, gpt-4o, etc.
  apiKey    String   // 加密存储
  baseUrl   String?  // 自定义端点（可选）
  isDefault Boolean  @default(false)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  @@unique([provider, model])
}
```

**API 端点**：

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/ai-config` | 获取所有配置 |
| GET | `/api/ai-config/default` | 获取当前默认配置 |
| POST | `/api/ai-config` | 创建配置 |
| PUT | `/api/ai-config/:id` | 更新配置 |
| DELETE | `/api/ai-config/:id` | 删除配置 |
| POST | `/api/ai-config/:id/test` | 测试连接 |

**LlmInvokeService 改动**：

读取 API Key 优先级：
1. 请求头 `X-LLM-Provider` + `X-LLM-API-Key`（前端指定）
2. 数据库 `AIConfig` 表中 `isDefault=true` 的配置
3. `.env` 环境变量（兼容私有部署）

**安全性**：
- API Key 在数据库中加密存储（AES-256）
- 前端不存储 Key，只存储 provider + model
- 传输使用 HTTPS

## 七、风险与约束

| 风险 | 缓解措施 |
|------|----------|
| 全面重构工作量大 | 分阶段交付：先基础设施，再核心页面 |
| API Key 安全 | 数据库加密存储（AES-256），前端不持有 Key |
| 现有功能回归 | 保留 .env 兼容，数据库无配置时回退到环境变量 |
| 流式输出兼容 | 测试所有 LLM provider 的流式支持 |
| 加密密钥管理 | 加密密钥通过环境变量 `AI_CONFIG_SECRET` 提供 |
