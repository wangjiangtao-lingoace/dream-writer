- Modify: `server/src/routes/ai.ts:67-71`

**问题分析:**
正文生成使用"模拟流式"——先调用 LLM 生成完整内容，再分块发送给前端。

- [ ] **Step 1: 检查当前实现**

```typescript
// server/src/routes/ai.ts:67-71
// 当前代码：先生成完整内容，再分块发送
const fullContent = await llmService.generate(prompt);

  res.write(fullContent.slice(i, i + 50));
}
```

- [ ] **Step 2: 修改为真正流式**

```typescript
// 使用流式 API
const stream = await llmService.streamGenerate(prompt);
for await (const chunk of stream) {
  res.write(chunk);
}
```

如果没有 `streamGenerate` 方法，需要在 `LlmInvokeService` 中添加。

### Task 6: 优化批量生成并行执行

**Files:**
- Modify: `server/src/routes/batchGenerate.ts:46`

**问题分析:**
批量生成使用 `for` 循环串行执行，耗时极长。

- [ ] **Step 1: 检查当前实现**

```typescript
// server/src/routes/batchGenerate.ts:46
// 当前代码：串行执行
for (const chapter of chapters) {
  await generateChapter(chapter);
}
```

- [ ] **Step 2: 修改为并行执行**

```typescript
// 使用 Promise.all 并行执行
await Promise.all(
  chapters.map(chapter => generateChapter(chapter))
);
```

- [ ] **Step 3: 添加并发控制**

避免同时发起太多 LLM 调用：

```typescript
// 限制并发数为 3
const concurrencyLimit = 3;
const results = [];
for (let i = 0; i < chapters.length; i += concurrencyLimit) {
  const batch = chapters.slice(i, i + concurrencyLimit);
  const batchResults = await Promise.all(
    batch.map(chapter => generateChapter(chapter))
  );
  results.push(...batchResults);
}
```

### Task 7: 修复自动生成状态持久化

**Files:**
- Modify: `server/src/services/AutoGenerateService.ts:43`

**问题分析:**
全局状态存储在内存 Map 中，进程重启后丢失。

- [ ] **Step 1: 检查当前实现**

```typescript
// server/src/services/AutoGenerateService.ts:43
// 当前代码：内存存储
const autoGenerateStatuses = new Map<string, AutoGenerateStatus>();
```

- [ ] **Step 2: 方案选择**

**方案 A: 使用数据库持久化**
- 创建 `AutoGenerateStatus` 表
- 修改 service 使用 Prisma 操作

**方案 B: 使用 Redis**
- 需要引入 Redis 依赖
- 适合分布式场景

**方案 C: 使用文件持久化**
- 简单，写入 JSON 文件
- 适合单机场景

建议选择 **方案 A**，使用数据库持久化。

- [ ] **Step 3: 添加 Prisma Schema**

```prisma
model AutoGenerateStatus {
  id        String   @id @default(cuid())
  novelId   String   @unique
  status    String   @default("idle") // idle, running, completed, failed
  progress  Int      @default(0)
  error     String?
  startedAt DateTime?
  updatedAt DateTime @updatedAt

  @@index([novelId])
}
```

- [ ] **Step 4: 修改 Service 使用数据库**

```typescript
// 替换内存 Map 为数据库操作
async getStatus(novelId: string): Promise<AutoGenerateStatus | null> {
  return prisma.autoGenerateStatus.findUnique({
    where: { novelId },
  });
}

async updateStatus(novelId: string, data: Partial<AutoGenerateStatus>): Promise<void> {
  await prisma.autoGenerateStatus.upsert({
    where: { novelId },
    update: data,
    create: { novelId, ...data },
  });
}
```
