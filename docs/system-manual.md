# Dream Writer 系统说明书

## 第一部分：系统概述

### Dream Writer 是什么

Dream Writer 是一个 AI 辅助的长篇小说创作平台。它帮助作者从零开始创作一部完整的小说，也可以分析已有作品的写作技巧并用于仿写创作。

### 核心定位

- **AI 长篇创作工作台**：不是简单的文字生成器，而是一套覆盖"灵感 → 大纲 → 人物 → 世界观 → 章节 → 校验"全流程的创作系统
- **三种创作模式**：自主创作、拆书分析、导入续写
- **13 个工作台模块**：总控、大纲、世界观、人物、风格、卷纲、主线、钩子、写作、拆书、知识库、记忆、校验

### 技术架构简述

系统采用前后端分离架构：

- **前端**（浏览器界面）：React + Vite，负责展示页面和用户交互
- **后端**（服务器）：Express + SQLite，负责数据存储和 AI 调用
- **AI 层**：支持多种大模型（DeepSeek、OpenAI、Anthropic、通义千问等），用户可自行配置 API Key

## 第二部分：快速开始

### 首次使用操作步骤

1. **启动系统**：运行 `pnpm dev`，浏览器访问 `http://localhost:5173`
2. **配置 AI 模型**：首次进入时，书架页面顶部会显示"配置 AI 模型开始创作"横幅，点击"立即配置"进入设置页，填写你的 API Key
3. **选择创作方式**：点击"新建作品"，选择三种创作方式之一
4. **开始创作**：进入工作台后，即可使用所有创作功能

### 从零开始创作一部小说的完整流程

1. 点击左侧菜单"创建新作品"
2. 选择"独立创作"
3. 填写作品标题、类型、创作灵感（越详细越好）
4. 点击"开始创作"，系统自动创建作品并进入工作台
5. 在工作台"总控"页面查看创作进度，点击"续写下一章"按钮
6. AI 自动生成大纲、人物、世界观、章节内容
7. 在"写作"页面查看和编辑每个章节
8. 使用"校验"功能检查一致性问题

## 第三部分：页面详解

### 3.1 书架页面（首页）

**功能说明**：展示用户所有已创建的小说作品，是系统的入口页面。

**界面布局**：
- 顶部：页面标题"我的作品" + "新建作品"按钮
- 未配置 API Key 时：顶部显示配置提醒横幅
- 主区域：作品卡片列表，每张卡片显示封面、标题、类型、章节数、字数、更新时间、状态标签（进行中/已完成/已暂停）

**操作步骤**：
1. 点击作品卡片进入该作品的工作台
2. 点击"新建作品"进入创作方式选择页
3. 点击卡片右侧的删除图标可删除作品（需二次确认）

### 3.2 创建方式选择页

**功能说明**：让用户选择三种创作路径之一。

**界面布局**：三张大卡片并排展示，每张卡片包含图标、标题、描述和功能特点列表。

**操作步骤**：
1. 点击"独立创作" → 进入自主创作表单
2. 点击"拆书创作" → 进入拆书分析页
3. 点击"导入续写" → 进入导入续写页

### 3.3 自主创作表单

**功能说明**：填写新作品的基本信息，由 AI 根据灵感自动生成大纲和后续内容。

**界面布局**：
- 左侧：封面上传区域（280x380，支持 JPG/PNG）
- 右侧：表单区域，包含标题、类型（下拉选择）、创作灵感（大文本框）

**操作步骤**：
1. 点击封面区域上传封面图片（可选）
2. 输入作品标题（必填，最多50字）
3. 选择作品类型（都市玄幻、仙侠修真、历史架空等）
4. 填写创作灵感（支持粘贴完整大纲、人物设定、世界观等，AI 会智能识别并保留原创内容）
5. 点击"开始创作"

### 3.4 拆书分析页

**功能说明**：分析已有作品的写作技巧和结构，用于仿写创作。

**操作步骤**：
1. 输入要分析的作品标题
2. 可选择：粘贴原文内容（至少80字，最多12000字），或上传文件（支持 txt/pdf/epub），或让系统自动搜索
3. 点击"搜索作品内容"或"直接拆书"
4. 预览搜索结果（标题、简介、大纲、人物）
5. 点击"开始拆解分析"，系统自动完成拆书
6. 分析完成后点击"创建新作品"

### 3.5 导入续写页

**功能说明**：上传已完成的小说文本，自动提取人物、世界观、剧情等信息后继续创作。

**操作步骤**：
1. 填写作品标题和类型（可选）
2. 上传 .txt 文件或直接粘贴文本（至少100字）
3. 点击"开始分析"
4. 系统自动提取人物、世界观、剧情和风格
5. 分析完成后自动跳转到工作台

### 3.6 工作台 - 写作页面（三栏布局）

**功能说明**：核心创作界面，采用左-中-右三栏布局。

**界面布局**：

**顶部栏**：
- 左侧：作品标题 + "返回书架"按钮
- 中间：今日目标、完成进度条、预计耗时、连续创作天数
- 右侧：当前情绪、节奏、高潮状态标签

**左栏 - 章节侧栏**（280px）：
- 按卷分组的章节列表，每章显示序号、标题、字数、情绪标签、冲突标签
- 顶部有"续写"按钮（AI 自动生成下一章）和"新建章节"按钮

**中栏 - 编辑区**：
- 章节头：面包屑导航、章节标题、字数统计、本章目标、核心情绪、读者风险
- 富文本编辑器：支持粗体、标题、伏笔标记（黄色下划线）、角色引用（蓝色高亮）
- 工具栏按钮："去AI味"、"增强压迫"
- 编辑内容自动保存（2秒防抖）

**右栏 - 资产面板**（340px）：
- 四个标签页：AI编辑（本章评分和建议）、角色（人物卡片列表）、世界观、伏笔

**底部栏**：
- 左侧：AI 分析文字
- 中间：三个雷达条 - 爽点密度、情绪波动、信息释放
- 右侧：下一步建议

### 3.7 工作台 - 其他页面

其他页面（大纲、世界观、人物等）采用旧版布局：顶部标题栏 + 左侧功能导航 + 中间内容区 + 右侧 AI 面板。

## 第四部分：三大创作模式详解

### 4.1 自主创作

**完整流程**：
1. 书架 → 新建作品 → 独立创作
2. 填写标题、类型、创作灵感
3. 系统创建作品，进入工作台
4. 工作台总控页显示流程节点：灵感 → AI 规划 → 章节创作
5. 点击"续写下一章"，AI 自动生成大纲 + 正文
6. 在写作页面编辑和润色内容
7. 使用校验功能检查一致性

**特点**：全程 AI 驱动，用户只需提供灵感即可。

### 4.2 拆书分析

**完整流程**：
1. 书架 → 新建作品 → 拆书创作
2. 输入参考作品标题，或粘贴/上传原文
3. 系统搜索并分析原作品（8 个维度：结构、风格、人物、世界观等）
4. 查看拆书结果，可修改每个分区
5. 点击"创建新作品"，系统基于分析结果生成新作品设定
6. 进入工作台后，可继续生成仿写方案（蓝图 + 章节模板 + 样章）
7. 点击"自动仿写 1-3 章"启动自动创作

**特点**：学习已有作品的技巧，生成风格相似的原创作品。

### 4.3 导入续写

**完整流程**：
1. 书架 → 新建作品 → 导入续写
2. 上传或粘贴已完成的小说文本（至少100字）
3. 系统自动提取：人物、世界观、剧情摘要、写作风格
4. 自动拆分章节
5. 进入工作台，直接从已有内容后续写

**特点**：适合已有部分作品、希望 AI 帮忙续写的场景。

## 第六部分：AI 能力说明

### 6.1 AI 生成章节

- **触发方式**：写作页面点击"AI生成"按钮，或工作台"续写下一章"
- **工作原理**：AI 根据已有大纲、人物、世界观、前文内容、风格配置，自动生成章节正文
- **输出**：直接写入编辑器，支持流式输出（逐字显示）

### 6.2 AI 续写

- **触发方式**：章节侧栏"续"按钮，或总控台"续写下一章"
- **工作原理**：基于前文上下文和记忆系统，自动生成下一章的大纲和正文
- **输出**：自动创建新章节，更新记忆和剧情状态

### 6.3 AI 拆书分析

- **触发方式**：拆书面板"一键拆书并创作"或"只创建拆书"
- **工作原理**：将参考文本分为 8 个维度分析（结构、风格、人物塑造、世界观构建等），每个分区独立生成分析结果
- **输出**：8 个分区的分析内容，可修改后生成仿写方案（蓝图 + 章节模板 + 样章）

### 6.4 AI 对话助手

- **触发方式**：工作台右侧面板的 AI 面板
- **工作原理**：根据当前页面上下文提供不同的 AI 操作（大纲页可生成大纲，人物页可生成人物卡，写作页可续写/检查/润色）
- **可用操作**：分析进度、生成大纲、生成人物卡、续写本章、一致性检查、润色优化

### 6.5 AI 流水线

- **触发方式**：总控台启动自动创作，或拆书面板"自动仿写 1-3 章"
- **工作原理**：自动化执行多个创作步骤（规划 → 结构化 → 写作 → 质量检查），每个步骤可暂停、确认、重新生成
- **输出**：自动生成大纲、世界观、人物、风格、卷纲、章纲、主线、钩子、章节正文

### 6.6 其他 AI 功能

- **AI 生成卷纲/章纲**：卷纲编辑器中的"AI 生成"按钮
- **AI 生成主线/钩子**：主线和钩子面板中的"AI 生成"按钮
- **从文本学习风格**：风格面板中粘贴参考文本，AI 自动提取风格特征
- **去 AI 味**：风格面板中的测试功能，或写作工具栏的"去AI味"按钮
- **一致性校验**：校验面板中选择章节，AI 检查战力、人设、世界观等一致性问题

## 第七部分：数据模型

### 核心实体关系

```
小说 (Novel)
  ├── 章节 (Chapter)          — 1:N，按顺序排列
  ├── 人物 (Character)        — 1:N，同一作品下人物名唯一
  ├── 世界观 (Worldview)      — 1:N，包含基础设定/力量体系/地理/历史
  ├── 卷纲 (Volume)           — 1:N，按排序排列
  │     └── 章纲 (ChapterOutline) — 1:N，每卷包含多个章纲
  ├── 知识资产 (KnowledgeAsset) — 1:N，设定/灵感/大纲/笔记/参考
  ├── 记忆 (Memory)           — 1:N，六类记忆（世界/角色/剧情/伏笔/爽点/风格）
  ├── 风格配置 (StyleProfile)  — 1:N，可设多个风格，一个为默认
  ├── 剧情状态 (StoryState)    — 1:1，跟踪当前进度/情绪/爽点/伏笔
  ├── 主线 (Mainline)         — 1:N，主线/支线/情感线/悬念线
  ├── 钩子 (Hook)             — 1:N，悬念/伏笔/喜剧等类型
  ├── 伏笔 (Foreshadow)       — 1:N，已埋/已收/过期状态
  ├── 爽点 (PleasurePoint)    — 1:N，记录每章的爽点
  ├── 情绪曲线 (EmotionCurve) — 1:N，记录每章情绪数据
  ├── 一致性问题 (ConsistencyIssue) — 1:N
  └── 流水线任务 (PipelineJob) — 1:1
        └── 阶段结果 (PhaseResult) — 1:N
```

### 关键关系说明

- **小说 → 章节**：一部小说包含多个章节，章节按顺序排列
- **小说 → 卷纲 → 章纲**：小说分卷，每卷包含章纲（规划用），与实际章节独立
- **小说 → 人物**：人物绑定到小说，同一小说下人物名不能重复
- **拆书分析 → 仿写方案**：拆书结果可生成仿写方案，仿写方案可启动自动创作
- **记忆系统**：自动记录创作过程中的关键信息，分为永久/长期/短期/临时四个层级

## 第八部分：API 端点清单

### 小说管理 (`/api/novels`)

| 方法 | 路径 | 用途 |
|------|------|------|
| GET | `/api/novels` | 获取所有小说列表 |
| POST | `/api/novels` | 创建新小说 |
| GET | `/api/novels/:id` | 获取小说详情（含章节） |
| PUT | `/api/novels/:id` | 更新小说信息 |
| DELETE | `/api/novels/:id` | 删除小说 |
| GET | `/api/novels/:id/workflow-status` | 获取工作流状态（总控台用） |
| GET | `/api/novels/:id/workspace-data` | 获取工作台数据 |
